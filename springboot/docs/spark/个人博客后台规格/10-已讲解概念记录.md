# 10-已讲解概念记录

本文用于记录学习过程中已经讲过的概念。后续拆任务、继续讲解或回顾项目时，先读取本文件，再决定这次只补哪些新知识，避免把已经讲过的内容重复讲一遍。

## 已讲解概念

- DTO / VO / Entity 的职责区别：Entity 对数据库，DTO 接请求参数，VO 返回前端响应数据。
- 登录接口不直接返回 `Admin` 实体，避免返回 `password`、`passwordVersion` 等内部字段。
- `@NotBlank` 用于字符串非空校验，适合用户名、密码这类字段。
- MyBatis 下划线字段到 Java 驼峰字段可以通过 `map-underscore-to-camel-case` 全局配置处理。
- `@Param` 用于让 MyBatis XML 稳定引用 Mapper 方法参数。
- `password_version` 用于密码版本控制，修改密码后递增，让旧 Token 失效。
- `password_version` 不自增，不唯一，不做主键。
- 业务日志使用 `@Slf4j` 输出，不能记录密码、Token、数据库口令等敏感信息。
- MyBatis SQL 日志继续走 `mybatis.configuration.log-impl`，不写在 `Mapper.xml`，不由 Logback 接管。
- Logback 当前只负责业务控制台日志，不做日志文件、滚动日志、操作日志表或通用日志 AOP。
- `JwtUtils` 中 Token 已讲过 `adminId`、`username`、`passwordVersion` 三类数据的作用。
- `PasswordEncoder` 和 `BCryptPasswordEncoder` 用于 BCrypt 密码加密与校验，只需 `spring-security-crypto`，不需要完整 `spring-boot-starter-security`。
- 登录失败统一返回“用户名或密码错误”，避免暴露后台账号是否存在。
- Controller 类级 `@RequestMapping` 只放模块公共前缀，例如 `/api/admin`；`/login`、`/profile`、`/validate` 等具体动作路径放在方法级注解上。

- `@RequestHeader` 用于在 Controller 方法参数中读取请求头，适合接收 `Authorization` 这类不是请求体的数据。

- `HandlerInterceptor` 用于在请求进入 Controller 前做统一拦截，适合 JWT 登录校验这类横向入口判断。

- `WebMvcConfigurer` 用于注册 Spring MVC 配置，例如把自定义拦截器挂到指定路径并配置白名单。

- `request.setAttribute` 可以把拦截器解析出的当前登录信息临时放到本次请求里，后续 Controller/Service 只在本次请求周期内读取。

- JWT 无状态退出表示后端不保存登录会话，退出登录由前端清理 Token；后端退出接口只用于走一次登录校验并返回成功。

- `@JsonInclude(JsonInclude.Include.NON_NULL)` 用于让 Jackson 序列化响应对象时跳过 null 字段，适合复用同一个 VO 但不同接口返回字段略有差异的场景。

- `AdminVO` 作为管理员相关接口统一响应对象，登录时带 `token`，当前管理员和资料接口不设置 `token`。

- Mapper 更新方法通常返回影响行数，Service 可以用影响行数判断本次更新是否真的命中了数据。

- `Map.of(...)` 可用于返回字段很少的一次性响应对象，例如 `{valid:true}` 或 `{token:"..."}`；复杂或复用响应仍应定义 VO。

- Maven 多模块用于把一个工程拆成父工程和多个子模块；父工程管理版本与模块列表，子模块承载具体代码与依赖。

- 父工程 `packaging=pom` 表示它主要负责聚合模块和管理配置，本身不产出可运行 jar。

- 按功能模块拆分表示 Maven 子模块优先对应 admin、article、comment 等业务域；每个功能模块内部继续使用 controller、service、mapper 等分层包。

- 模块依赖方向必须单向清晰，启动模块依赖功能模块，功能模块依赖 common，common 不能反向依赖任何功能模块。

- IDEA 的 Maven 工具窗口用于重新加载 Maven 工程、查看模块依赖和执行生命周期目标，是学习多模块工程时的主要操作入口。

- Maven 生命周期中的 `compile`、`test`、`package` 分别用于编译、运行测试和打包；多模块项目应优先从父工程执行，验证模块聚合是否正确。

- 主键用于唯一标识一行数据，文章表的 `id` 适合作为主键并使用自增生成。

- 非空约束用于保证核心字段必须有值，例如文章标题、正文、状态、创建时间和更新时间。

- 默认值用于让数据库在插入数据时自动补齐常见初始值，例如布尔标记默认 `false`、计数字段默认 `0`。

- 取值约束用于限制状态字段只能落在业务允许范围内，例如文章状态只能是 `draft`、`published`、`hidden`。

- 外键关系用于表达表与表之间的关联；本项目为单管理员后台，文章表不保存 `admin_id`，分类和标签关系本阶段暂不落地。

- 索引用于提升常见筛选和排序查询效率，文章表后续需要围绕状态、创建时间、发布时间建立查询索引。

- `INSERT` 用于向数据库表新增记录，示例数据通常只写业务字段，不手动写自增主键。

- 示例数据用于覆盖核心业务状态和查询场景，本阶段文章主表至少要覆盖草稿、已发布、隐藏三种状态。

- Java 的 `Long` 适合承接数据库 `bigint` 字段，例如文章 ID、分类 ID 这类可能增长较大的主键。

- Java 的 `OffsetDateTime` 适合承接 PostgreSQL `timestamptz`，能保留日期、时间和时区偏移信息。

- `@Size` 用于限制字符串、集合等长度范围，适合文章标题、摘要、正文这类有长度边界的请求字段。

- `@Pattern` 用于限制字符串必须符合指定正则规则，适合文章状态只能提交 `draft`、`published`、`hidden` 的场景。

- `@NotNull` 用于限制包装类型或对象不能为 null，适合布尔值、数字 ID 等不能用 `@NotBlank` 判断的字段。

- Lombok 的 `@Builder` 用于用链式方式组装对象，适合 VO 从 Entity 转换时清楚地指定每个返回字段。

- VO 中的 `from(Entity)` 静态转换方法用于把数据库实体转换成前端响应对象，避免 Controller 直接返回 Entity。

- Mapper 接口用于声明数据库访问能力，Service 调用 Mapper 方法，不直接写 SQL。

- MyBatis XML 的 `<select>` 用于编写查询 SQL，通过 `id` 和 Mapper 接口方法名对应。

- `resultType` 用于告诉 MyBatis 查询结果要封装成哪个 Java 类型，适合字段名和 Java 属性能正常映射的简单查询。

- `@GetMapping` 用于声明 GET 查询接口，适合文章详情、列表这类读取数据但不修改数据的请求。

- `@PathVariable` 用于从 URL 路径中读取变量，例如 `/articles/{id}` 中的文章 ID。

- `@PostMapping` 用于声明 POST 新增接口，适合创建文章这类会新增数据库记录的请求。

- `@RequestBody` 用于把 JSON 请求体绑定成 DTO，适合新增、修改这类需要提交多字段数据的接口。

- `@Valid` 用于触发 DTO 参数校验，让 `@NotBlank`、`@Size`、`@Pattern` 等校验注解在进入业务逻辑前生效。

- MyBatis `useGeneratedKeys` 和 `keyProperty` 用于新增记录后把数据库生成的自增主键回填到 Entity 字段，适合新增文章后返回新文章 ID。

- `@PutMapping` 用于声明 PUT 修改接口，适合按 ID 整体更新文章这类已有资源。

- SQL 的 `UPDATE ... SET ... WHERE ...` 用于修改已有记录，必须带明确的 `WHERE` 条件，避免误更新整张表。

- `@DeleteMapping` 用于声明 DELETE 删除接口，适合按 ID 删除文章这类已有资源。

- SQL 的 `DELETE FROM ... WHERE ...` 用于删除已有记录，必须带明确的 `WHERE` 条件，避免误删整张表。

- 数据库分页通常用 `LIMIT` 控制每页条数，用 `OFFSET` 跳过前面页的数据，计算公式是 `offset = (page - 1) * size`。

- 分页查询必须配合稳定的 `ORDER BY`，否则数据库每次返回顺序可能不稳定，翻页时容易出现重复或遗漏。

- PageHelper 的 `PageHelper.startPage(page, size)` 用于在 MyBatis 查询前开启分页，必须紧挨着放在 Mapper 查询前面。

- `PageInfo` 用于承接 PageHelper 的分页结果，包含当前页数据、总条数、总页数等分页信息。

- `PageResult` 是项目自己的分页响应对象，用于把 `PageInfo` 转成前端更稳定的 `page、size、total、pages、list` 结构。

- MyBatis XML 的 `<sql>` 和 `<include>` 用于复用 SQL 片段，适合把文章列表和详情共用的字段列抽出来，避免每个查询重复写一大串字段。

- MyBatis 动态 SQL 的 `<if>` 根据 DTO 字段是否满足条件决定是否拼接对应 SQL，适合可选筛选条件，不能用 `${}` 直接拼接用户输入。

- MyBatis 动态 SQL 的 `<where>` 只在内部存在有效条件时生成 `WHERE`，并自动移除开头多余的 `AND` 或 `OR`，适合组合多个可选筛选条件。

- PostgreSQL 的 `generated by default as identity` 用于定义自增主键列，通常替代旧写法 `serial`，并且允许在特殊情况下手动指定主键值。

- `CHECK` 约束用于限制字段只能取满足条件的值，适合分类状态这类固定枚举范围字段。

- `UNIQUE (字段1, 字段2)` 组合唯一约束用于限制两列组合不能重复，适合文章和标签关联表避免同一篇文章重复绑定同一标签。

- 外键上的 `on delete restrict` 表示父表记录被子表引用时不允许删除，适合分类仍被文章使用时直接拦截删除。

- 外键上的 `on delete cascade` 表示删除父表记录时自动删除关联子表记录，适合文章删除后自动清理文章标签关联关系。

- `create table if not exists` 用于建表时避免表已存在直接报错，适合本地重复执行学习脚本时降低中断概率。

- 一对多关系表示一条父数据可以对应多条子数据，适合“一个分类下有多篇文章”，文章表通过 `category_id` 保存所属分类。

- 多对多关系需要通过中间关联表落地，适合“一篇文章有多个标签、一个标签也能属于多篇文章”，本项目用 `article_tag` 保存文章和标签的绑定关系。

- `JOIN ... ON ...` 用于把多张表按关联条件连接起来查询，适合文章详情或列表同时返回分类名称、标签信息。

- `SELECT DISTINCT` 用于去除查询结果中的重复行，适合多表关联后避免重复返回同一个标签或同一篇文章。

- `EXISTS` 子查询用于判断关联记录是否存在，适合文章列表按标签筛选时只判断文章是否绑定某标签，避免直接 JOIN article_tag 导致文章重复。

- 删除前占用校验用于在删除分类、标签这类基础数据前先检查是否仍被文章使用，避免破坏已有业务数据关系。

- 关联数据存在性校验用于在保存主数据前确认请求中的外键 ID 对应记录真实存在，避免写入指向不存在分类或标签的无效关系。

- `@Resource` 和 `@Autowired` 都可用于依赖注入；本项目优先沿用已有 `@Resource` 风格，按名称优先、按类型兜底注入 Service、Mapper 等 Bean。

- `@Transactional` 用于把 Service 公开业务方法划为事务边界，使资料更新后回查、删除前校验与删除、文章和标签关联写入等数据库操作一起提交或一起回滚。

- `rollbackFor = Exception.class` 用于让事务在受检异常和运行时异常下都回滚；事务方法不能吞掉异常，否则 Spring 会按正常返回提交事务。

- `SELECT ... FOR UPDATE` 会在事务内锁定命中的数据库记录，适合修改密码、删除分类或标签这类必须避免“校验通过后数据又被并发修改”的业务流程。

- 先删后插的关联同步策略用于维护多对多关系：修改主数据时先删除旧关联，再按本次请求重建新关联，并用事务保证旧关联不会被删完后写入失败。

- H2 内存数据库夹具用于在不连接开发库的前提下验证 Mapper SQL；联表测试必须在临时库中创建完整关系表、约束和隔离数据，测试结束后自动销毁。

- `@ImportAutoConfiguration` 可在 `@MybatisTest` 测试切片中补充加载指定自动配置，本项目用它启用 PageHelper 分页拦截器并验证分页总数。

- Bean Validation 可以把 `@NotNull`、`@Positive` 写在集合元素类型上，校验批量请求中的每一个 ID，而不只校验集合本身是否为空。

- MyBatis `<foreach>` 可以安全展开 ID 集合并配合 `IN` 完成批量 SQL，集合必须先校验非空，避免生成非法的空 `IN` 条件。

- SQL `LIKE` 配合 `concat('%', #{keyword}, '%')` 可以用安全参数完成包含式模糊匹配，适合按文章标题或摘要搜索关键词。

- 公开查询边界用于隔离前台和后台数据权限：公开 Controller、Service、Mapper 独立承接访客请求，并在 SQL 中固定公开状态条件，不能依赖前端传参过滤草稿或隐藏数据。

- 数据库唯一索引可以把“同一访客对同一文章只能点赞一次”变成数据库约束；Service 先做友好校验，同时捕获并发插入导致的唯一键冲突。

- HttpOnly Cookie 适合保存匿名访客标识：浏览器自动携带、前端脚本不可读取；后端只保存其哈希值，不把 Cookie 原文作为业务数据落库。

- 时间窗口限频通过按哈希后的 IP 查询最近时间段记录数实现；超过阈值返回等待秒数，防刷判断必须在后端完成，不能依赖按钮禁用或 localStorage。

- `update ... set count = count + 1` 是数据库侧原子计数更新，适合浏览量和点赞量递增；更新成功后再查询最新值返回，避免前端自行推算。

- 横切关注点是同时作用于多个业务方法、但不属于某一个核心业务职责的公共逻辑，例如操作日志、耗时统计和事务管理。

- AOP 目标对象是真正执行业务逻辑的 Bean，例如 `ArticleServiceImpl`；代理对象由 Spring 放在调用方和目标对象之间，用于附加切面或事务行为。

- Spring AOP 代理调用链要求外部调用先经过代理对象，再由代理调用目标方法；只有经过代理的调用，切面才有机会执行。

## 使用与维护规则

- 每次继续讲解前先读取本文件，优先跳过已经记录过的概念。
- 继续讲解时，先对照本文件判断这次是否出现了新的通用概念。
- 如果本次出现了新的通用概念，必须先在当次讲解里把概念讲清楚，再在同一次任务里把概念追加到本文件，不能拖到下一次补记。
- 新概念讲解不能只写一句定义，至少要讲清楚：概念是什么、解决什么问题、为什么现在讲、会放在哪层或哪个文件、常见用法是什么、不要怎么用。
- 追加到本文件时保持精炼，记录“概念名 + 一句话职责/约束”即可，不把整段教学内容原样贴进来。
- 只记录可复用概念，不记录一次性代码细节、临时排错过程或单次改动说明。
- 如果本次任务没有产生新的通用概念，就不要为了凑记录而追加内容。
