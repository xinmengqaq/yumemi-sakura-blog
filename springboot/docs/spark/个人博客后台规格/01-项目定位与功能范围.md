# 01-项目定位与功能范围

## 项目定位

本项目是一个国内个人日常空间型博客后台，不按国外传统个人博客的功能习惯设计。

核心目标是支撑一个人记录和管理自己的日常内容，包括长文、短动态、相册、留言、评论、友链和站点展示信息。后台只服务个人使用，不做多人协作、不做企业 CMS、不做内容营销系统。

## 用户范围

第一版只允许一个管理员账号。

管理员账号由你手动入库维护，后台第一版不提供注册接口。

## 核心模块

后台第一版包含以下模块：

| 模块 | 说明 |
|---|---|
| 管理员模块 | 登录验证、JWT 签发、Token 校验、退出登录、管理员资料维护 |
| 文章管理 | 管理长内容，例如日记、随笔、记录、总结 |
| 说说管理 | 管理短动态，例如心情、碎碎念、生活瞬间 |
| 相册管理 | 管理图片合集和照片 |
| 分类管理 | 主要服务文章 |
| 标签管理 | 主要服务文章，说说可选使用 |
| 评论管理 | 管理文章和说说下的评论 |
| 留言板管理 | 管理访客留言和管理员回复 |
| 友链管理 | 管理朋友博客、个人站链接和友链申请 |
| 站点配置 | 管理头像、昵称、签名、公告、背景图、关于我、备案号、社交链接 |
| 统计概览 | 展示内容数量、访问量、热门内容、最新互动 |
| 工程结构 | 使用 Maven 多模块组织启动模块、公共模块和业务模块 |

## 第一版明确不做

- 不做多管理员管理。
- 不做角色权限、菜单权限、按钮权限。
- 不做管理员注册接口，管理员数据手动入库。
- 不做微服务拆分，多模块只用于单体工程内的代码边界管理。
- 不做操作日志入库和后台审计查询。
- 不做企业 CMS 的页面搭建器。
- 不做订阅、邮件营销、支付、会员。
- 不做复杂数据分析平台。
- 不做视频、音频、网盘资源管理。
- 不做公开多人注册。

## 内容管理范围

文章、说说、相册是三个独立后台模块。

文章适合长内容，支持草稿、发布、隐藏、置顶、推荐、分类、标签。

说说适合短内容，支持发布、隐藏、置顶和多图。

相册适合图片合集，支持相册、照片、封面、排序、隐藏。

## 国内个人博客风格特征

后台应覆盖国内个人站常见能力：

- 留言板。
- 友链。
- 说说。
- 相册。
- 首页公告。
- 关于我。
- 头像、签名、社交链接。
- 评论审核和回复。
- 备案号配置。

这些能力比 newsletter、订阅营销、作者专栏等国外传统博客功能更适合本项目。
