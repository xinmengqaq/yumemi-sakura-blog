# 03-数据库表设计

本设计只描述字段、类型、约束和关系，不写 SQL。

类型按 PostgreSQL 习惯设计。

## 1. admin 管理员表

| 字段 | 类型 | 约束/说明 |
|---|---|---|
| id | bigint | 主键，自增 |
| username | varchar(255) | 当前库字段，登录账号 |
| password | varchar(255) | 当前库字段，存加密密码 |
| name | varchar(255) | 当前库字段，管理员名称 |
| role | varchar(255) | 当前库字段，当前项目只做单管理员标识，不展开多角色设计 |
| avatar | varchar(255) | 当前库字段，头像地址 |
| password_version | integer | 必填，默认 1，用于修改密码后让旧 Token 失效 |

说明：当前学习库里的 `admin` 表以现有实际字段为准，不额外扩展 `nickname`、`intro`、`email`、`created_at`、`updated_at` 等字段。

约束：业务上按单管理员场景使用，系统只维护一条管理员记录。

## 2. article 文章表

| 字段 | 类型 | 约束/说明 |
|---|---|---|
| id | bigint | 主键，自增 |
| category_id | bigint | 可空，对应接口 `categoryId`，关联 category.id |
| title | varchar(120) | 必填 |
| summary | varchar(300) | 可空 |
| content | text | 必填 |
| cover_url | varchar(500) | 可空 |
| status | varchar(20) | 必填：draft/published/hidden |
| is_top | boolean | 必填，默认 false |
| is_recommend | boolean | 必填，默认 false |
| view_count | integer | 必填，默认 0 |
| comment_count | integer | 必填，默认 0 |
| like_count | integer | 必填，默认 0 |
| published_at | timestamptz | 可空 |
| created_at | timestamptz | 必填 |
| updated_at | timestamptz | 必填 |

关系：本项目为单管理员后台，文章不保存 `admin_id`；分类一对多文章；文章和标签多对多。接口响应中的 `categoryName` 来自 category.name 联表查询，不是 article 表字段。

## 2.1 article_like 匿名文章点赞记录表

| 字段 | 类型 | 约束/说明 |
|---|---|---|
| id | bigint | 主键，自增 |
| article_id | bigint | 必填，关联 article.id，文章删除时级联删除 |
| visitor_key_hash | char(64) | 必填，匿名访客 Cookie 标识的 SHA-256 哈希 |
| ip_hash | char(64) | 必填，请求来源 IP 的 SHA-256 哈希，用于时间窗口限频 |
| created_at | timestamptz | 必填，默认当前时间 |

约束：`article_id + visitor_key_hash` 唯一，同一匿名访客对同一文章只能点赞一次；数据库只保存哈希值，不保存原始 Cookie 或 IP。

## 5. article_tag 文章标签关联表

| 字段 | 类型 | 约束/说明 |
|---|---|---|
| article_id | bigint | 必填，关联 article.id |
| tag_id | bigint | 必填，关联 tag.id |

约束：`article_id + tag_id` 组合唯一。

接口映射：文章新增和修改接口中的 `tagIds` 数组落库到本表，一篇文章对应多条 `article_id + tag_id` 记录；`tagIds` 不是 article 表字段。

## 7. moment_image 说说图片表

| 字段 | 类型 | 约束/说明 |
|---|---|---|
| id | bigint | 主键，自增 |

## 覆盖的数据库学习点

- 一对多：分类和文章、相册和照片、说说和图片。
- 多对多：文章和标签。
- 自关联：评论回复评论。
- 状态流转：文章、评论、留言、友链。
- 排序：分类、相册、照片、友链。
- 统计更新：阅读量、评论数、访问量。
- 唯一约束：用户名、分类名、标签名、友链地址、访问日期。
- 删除规则：删除前检查关联，或同步清理关联数据。
