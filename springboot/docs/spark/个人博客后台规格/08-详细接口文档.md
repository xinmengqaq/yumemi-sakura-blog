# 08-详细接口文档

本文按国内后台项目常见写法编写。每个接口都包含：接口名称、接口说明、请求 URL、请求方法、请求参数、请求示例、响应结果、响应示例、错误码及含义、接口版本。

## 1. 公共约定

### 1.1 基础信息

接口版本：`v1`

后台接口前缀：`/api/admin`

请求数据格式：`application/json`

后台接口默认需要登录。登录、Swagger 文档、静态资源除外。

登录后请求头：

```http
Authorization: Bearer <token>
```

### 1.2 统一响应格式

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {}
}
```

失败示例：

```json
{
  "code": "401",
  "msg": "登录已过期，请重新登录",
  "data": null
}
```

### 1.3 分页响应格式

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {
    "page": 1,
    "size": 10,
    "total": 36,
    "pages": 4,
    "list": []
  }
}
```

分页参数：

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|---|---|---|---|---|
| page | integer | 否 | 1 | 当前页，从 1 开始 |
| size | integer | 否 | 10 | 每页数量，最大 100 |
| keyword | string | 否 | 空 | 关键词 |
| status | string | 否 | 空 | 状态 |
| sort | string | 否 | createdAt | 排序字段，后端白名单控制 |
| order | string | 否 | desc | asc 或 desc |

说明：

- 分页参数只用于集合型资源列表接口。
- 管理员模块为单人后台，不提供管理员列表分页接口。

### 1.4 批量操作约定

批量操作用于后台列表页的多选处理，第一版只做批量删除，不做批量修改状态、批量置顶、批量推荐等容易误操作的接口。

批量删除统一使用 `POST` 方法，路径统一为当前资源集合路径下的 `/batch-delete`，请求体统一接收 `ids` 数组。

请求示例：

```json

{

  "ids": [1, 2, 3]

}

```

响应结果统一返回实际删除数量：

```json

{

  "code": "200",

  "msg": "删除成功",

  "data": {

    "deletedCount": 3

  }

}

```

规则说明：

- `ids` 必填，不能为空数组。
- `ids` 中的 ID 必须去重后再执行业务处理。
- 批量删除必须复用单条删除的业务规则，例如文章要清理标签关联和评论。
- 不要求因为某个 ID 不存在就整批失败；返回值以实际删除成功数量为准。
- 如果存在业务规则禁止删除的数据，可以整批返回业务错误，避免出现用户误以为全部删除成功的情况。

## 2. 管理员登录验证接口

### 2.1 管理员登录

接口名称：管理员登录

接口说明：校验用户名和密码，登录成功后返回 JWT 和管理员基础信息。

请求 URL：`/api/admin/login`

请求方法：`POST`

请求参数：

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|---|---|---|---|---|
| username | string | 是 | 无 | 用户名 |
| password | string | 是 | 无 | 密码 |

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

响应结果：

| 字段 | 类型 | 描述 |
|---|---|---|
| id | long | 管理员 ID |
| username | string | 用户名 |
| name | string | 管理员名称 |
| role | string | 当前管理员角色标识 |
| avatar | string | 头像 |
| token | string | JWT |

响应示例：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {
    "id": 1,
    "username": "admin",
    "name": "梦梦",
    "role": "admin",
    "avatar": "/files/avatar.png",
    "token": "jwt-token"
  }
}
```

错误码及含义：

| 错误码 | 含义 |
|---|---|
| A002 | 用户名或密码错误 |
| 400 | 参数错误 |

接口版本：`v1`

## 4. 文章接口

### 4.1 分页查询文章

接口名称：分页查询文章

接口说明：后台文章列表，支持关键词、状态、分类、标签筛选。列表只返回列表展示字段；按标签筛选时必须保证同一篇文章只出现一次，并保持分页统计准确。

请求 URL：`/api/admin/articles`

请求方法：`GET`

请求参数：

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|---|---|---|---|---|
| page | integer | 否 | 1 | 当前页 |
| size | integer | 否 | 10 | 每页数量 |
| keyword | string | 否 | 空 | 标题或摘要关键词 |
| status | string | 否 | 空 | draft、published、hidden |
| categoryId | long | 否 | 空 | 分类 ID |
| tagId | long | 否 | 空 | 标签 ID |

请求示例：

```http
GET /api/admin/articles?page=1&size=10&status=published&keyword=日常
Authorization: Bearer jwt-token
```

响应结果：

| 字段 | 类型 | 描述 |
|---|---|---|
| list[].id | long | 文章 ID |
| list[].title | string | 标题 |
| list[].summary | string | 摘要 |
| list[].coverUrl | string | 封面 |
| list[].categoryName | string | 分类名称 |
| list[].status | string | 状态 |
| list[].isTop | boolean | 是否置顶 |
| list[].isRecommend | boolean | 是否推荐 |
| list[].viewCount | integer | 阅读量 |
| list[].commentCount | integer | 评论数 |

响应示例：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {
    "page": 1,
    "size": 10,
    "total": 1,
    "pages": 1,
    "list": [
      {
        "id": 1,
        "title": "周末记录",
        "summary": "一些日常",
        "coverUrl": "/files/a.jpg",

        "isTop": true,

        "createdAt": "2026-07-03T09:00:00+08:00",
        "updatedAt": "2026-07-03T10:00:00+08:00"
      }
    ]
  }
}
```

错误码及含义：

| 错误码 | 含义 |

| 400 | 分页或状态参数错误 |

## 14. 前台公开内容接口

本节接口默认公开访问，不携带管理员 Token，不得复用或放开 `/api/admin/**`。所有文章查询只返回 `status = published` 的文章。文章列表和详情额外返回 `likeCount`，匿名点赞使用 `article_like` 记录防重复并同步更新 `article.like_count`。

### 14.1 查询首页内容

请求 URL：`/api/home`

请求方法：`GET`

响应结果：返回精选文章和最新文章。精选文章固定按 `is_recommend = true`、置顶优先、发布时间倒序查询；最新文章按置顶优先、发布时间倒序查询。两组数据均只包含已发布文章，数量由后端控制，数据不足时按实际数量返回。

### 14.2 分页查询公开文章

请求 URL：`/api/articles`

请求方法：`GET`

请求参数：`page`、`size`、`keyword`、`categoryId`、`tagIds`、`year`、`month`，均可选并支持组合；`tagIds` 使用逗号分隔的标签 ID，多个标签使用 OR 语义；`month` 必须与 `year` 一起使用。关键词匹配标题和摘要，标签筛选使用 `exists` 避免文章重复。

响应结果：使用统一 `PageResult`，列表项返回 `id`、`title`、`summary`、`coverUrl`、`categoryId`、`categoryName`、`isTop`、`isRecommend`、`viewCount`、`commentCount`、`likeCount`、`publishedAt`。排序固定为置顶优先、发布时间倒序、文章 ID 倒序。

### 14.3 查询公开文章详情

请求 URL：`/api/articles/{id}`

请求方法：`GET`

响应结果：在公开文章列表字段基础上返回 `content` 和 `tags`。文章不存在、草稿或隐藏时统一返回“文章不存在”，不返回残缺正文。文章确认公开后，当前请求将 `viewCount` 原子递增 1，并返回递增后的文章详情。

### 14.4 查询公开分类

请求 URL：`/api/categories`

请求方法：`GET`

响应结果：只返回可见分类的 `id`、`name`、`articleCount`，按 `sortOrder`、分类 ID 排序；`articleCount` 只统计已发布文章。

### 14.5 查询公开标签

请求 URL：`/api/tags`

请求方法：`GET`

响应结果：只返回至少关联一篇已发布文章的标签，字段为 `id`、`name`、`articleCount`，按标签 ID 倒序；数量只统计已发布文章。

### 14.6 查询文章归档元数据

请求 URL：`/api/articles/meta`

请求方法：`GET`

响应结果：按 `published_at` 聚合已有已发布文章，返回存在文章的年份、月份和文章数量；年份和月份均倒序，不生成空月份。

### 14.7 匿名点赞文章

请求 URL：`/api/articles/{id}/like`

请求方法：`POST`

请求体：空。

响应结果：返回递增后的 `likeCount`。后端通过 HttpOnly 匿名访客 Cookie 识别访客，只保存 Cookie 标识和来源 IP 的 SHA-256 哈希；同一访客对同一文章重复点赞返回 `409`，同一 IP 在 60 秒内最多点赞 10 次，超过后返回 `429`，并在 `data.retryAfterSeconds` 中返回等待秒数。文章不存在、草稿或隐藏时返回“文章不存在”。
