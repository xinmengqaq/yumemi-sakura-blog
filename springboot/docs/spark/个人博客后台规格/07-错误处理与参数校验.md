# 07-错误处理与参数校验

## 统一返回结构

继续沿用当前项目已有的 `Result` 思路，并固定格式。

成功：

```text
code: "200"
msg: "请求成功"
data: 具体数据
```

失败：

```text
code: 错误码
msg: 错误说明
data: null
```

分页成功：

```text
code: "200"
msg: "请求成功"
data:
  page: 当前页
  size: 每页数量
  total: 总条数
  pages: 总页数
  list: 数据列表
```

## 错误码分类

基础错误码：

| 错误码 | 含义 |
|---|---|
| 200 | 请求成功 |
| 400 | 参数错误 |
| 401 | 未登录或 Token 无效 |
| 403 | 无权限操作 |
| 404 | 数据不存在 |
| 409 | 数据冲突 |
| 500 | 系统异常 |

业务错误码：

| 错误码 | 含义 |
|---|---|
| A002 | 用户名或密码错误 |
| A003 | 旧密码错误 |
| A004 | Token 已过期 |
| B001 | 文章不存在 |
| B002 | 文章标题不能为空 |
| B003 | 文章正文不能为空 |
| C001 | 分类不存在 |
| C002 | 分类名称已存在 |
| C003 | 分类下存在文章，禁止删除 |
| T001 | 标签名称已存在 |
| M001 | 说说不存在 |
| P001 | 相册不存在 |
| P002 | 照片不存在 |
| R001 | 评论不存在 |
| R002 | 评论状态不允许这样修改 |
| L001 | 留言不存在 |
| F001 | 友链地址已存在 |

## 全局异常处理

建议新增：

```text
common/exception/BusinessException
common/exception/GlobalExceptionHandler
common/enums/ErrorCode
```

统一处理：

- 业务异常。
- 参数校验异常。
- JWT 异常。
- 数据不存在异常。
- 数据库异常。
- 未捕获异常。

返回规则：

- 业务异常返回业务错误码和明确提示。
- 参数异常返回 400。
- 未登录返回 401。
- 无权限返回 403。
- 数据不存在返回 404。
- 数据冲突返回 409。
- 未知异常返回 500，不把 Java 堆栈返回给前端。

## 参数校验分工

DTO 做基础格式校验。

Service 做业务校验。

DTO 负责：

- 是否为空。
- 字符长度。
- 邮箱格式。
- URL 格式。
- 数字范围。
- 图片数量上限。

Service 负责：

- 分类是否存在。
- 标签是否存在。
- 文章是否允许发布。
- 分类下有文章能不能删除。
- 管理员是否已经存在。
- 评论是否属于文章或说说。

## 常见字段校验

| 字段 | 校验 |
|---|---|
| username | 必填，3-50 个字符 |
| password | 必填，建议 6-50 个字符 |
| name | 管理员资料修改时必填，1-255 个字符 |
| avatar | 可空，最多 255 个字符 |
| nickname | 必填，1-50 个字符 |
| article.title | 必填，1-120 个字符 |
| article.summary | 最多 300 个字符 |
| article.content | 发布时必填 |
| moment.content | 必填，最多 2000 字 |
| album.name | 必填，1-80 个字符 |
| photo.description | 最多 200 个字符 |
| category.name | 必填，1-50 个字符 |
| tag.name | 必填，1-50 个字符 |
| comment.content | 必填，最多 1000 字 |
| message.content | 必填，最多 1000 字 |
| friend_link.site_url | 必填，必须是 URL |
| email | 可空，填写时必须是邮箱格式 |

## 分页参数校验

分页依赖 PageHelper 实现，不自研分页 SQL。

分页参数仍需校验：

| 参数 | 规则 |
|---|---|
| page | 默认 1，不能小于 1 |
| size | 默认 10，不能小于 1，不能大于 100 |
| keyword | 可空，做模糊搜索 |
| status | 可空，按状态筛选 |
| sort | 只能使用后端允许的排序字段 |
| order | 只能是 asc 或 desc |
| ids | 批量操作必填，不能为空数组，元素必须是合法 ID |

## 状态参数校验

状态值必须限制在固定范围。

| 模块 | 状态 |
|---|---|
| 文章 | draft / published / hidden |
| 说说 | published / hidden |
| 相册 | visible / hidden |
| 照片 | visible / hidden |
| 评论 | pending / approved / rejected |
| 留言 | pending / approved / rejected |
| 友链 | pending / approved / hidden / rejected |

非法状态直接返回参数错误。

## 删除校验

| 删除对象 | 校验规则 |
|---|---|
| 分类 | 分类下没有文章才允许删除 |
| 标签 | 没有关联文章才允许删除 |
| 文章 | 当前阶段删除文章标签关联；评论模块完成后再删除真实文章评论 |
| 说说 | 删除说说图片、说说评论 |
| 相册 | 删除相册照片 |
| 管理员 | 第一版不允许删除管理员 |

批量删除校验：

- 批量删除必须复用单条删除的业务校验和关联清理规则。

- `ids` 为空、缺失或元素不是合法 ID 时，直接返回参数错误。

- 文章、说说、评论这类删除后会影响关联数据或计数字段的资源，批量删除后必须同步处理关联数据。


## 日志配合

需要写 error 日志：

- 未捕获异常。
- 数据库异常。
- JWT 解析异常。

不需要写 error 日志：

- 参数为空。
- 用户名密码错误。
- 分类名称重复。
- 评论不存在。

这些属于正常业务失败，最多记 warn。

## 前端提示原则

错误提示要能指导处理。

推荐：

```text
文章标题不能为空
分类不存在
分类下存在文章，不能删除
登录已过期，请重新登录
```

不推荐：

```text
操作失败
系统错误
参数异常
```
