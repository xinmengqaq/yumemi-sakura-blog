# 01-前端项目定位与边界

## 项目定位

本项目是个人博客系统的前端工程，负责承载博客相关的前端界面。

前端工程后续可以同时包含两类界面：

- 前台展示界面：面向访客，用于浏览文章、分类、标签、说说、相册、友链等内容。
- 后台管理界面：面向管理员，用于登录后台并管理博客内容和站点配置。

当前阶段只规划整体前端工程规格，不展开具体前台页面或后台页面的业务设计。

前端项目使用 React + TypeScript + Vite 作为核心技术栈，主要由 AI 按规格完成交付。

用户主要专注后端学习和后端能力建设，不要求手写前端代码。前端工程的核心价值是提供可用的前台展示界面和后台管理界面，用于验证后端接口、展示后端数据、暴露后端错误和形成完整产品体验。

用户只需要理解前端和后端协作中的最小必要边界：接口如何被调用、数据如何展示、错误如何暴露、登录态如何影响请求、后端接口变更后前端哪些位置需要同步。

前端项目不承担后端业务逻辑，不直接访问数据库，不绕过后端接口操作数据。

## 项目目录位置

前端项目放在 Spring Boot 后端项目的上级目录，与后端项目保持同级。

```text
D:\daimai\项目学习\
├─ springboot\     # 后端项目
└─ blog-web\       # 前端项目
```

前端文档、源码、测试、构建产物都归属于 `blog-web`，不放入后端项目目录。

## 前后端关系

前端通过后端提供的 HTTP API 完成数据读写。

开发阶段，前端本地服务由 Vite 启动，后端本地服务由 Spring Boot 启动。前端请求统一经过前端请求层封装，再由开发服务器代理转发到后端服务。

```text
浏览器
  ↓
Vite 开发服务器
  ↓ API 代理
Spring Boot 后端服务
  ↓
数据库
```

前端不直接处理数据库连接、SQL、JWT 签名、密码加密、权限判断等后端职责。

## 当前规划范围

本轮文档只规划前端工程整体规格，包括：

- 技术栈选择。
- 目录结构。
- 路由组织。
- 请求封装。
- 接口类型。
- 状态管理。
- 本地缓存。
- 样式组织。
- 测试规范。
- 开发命令。
- 环境配置。

本轮文档不展开具体业务页面设计，不设计文章、分类、标签、评论、相册、文件、统计等模块的页面字段和交互细节。

前端开发顺序不单独设计，后续跟随后端开发阶段推进。

## 前台与后台边界

前台和后台属于同一个个人博客前端工程下的不同应用域。

前台界面重点关注内容展示、阅读体验、公开访问和搜索浏览。

后台界面重点关注登录态、内容管理、表单操作、列表筛选、状态变更和管理效率。

二者可以共享基础能力，例如请求封装、类型定义、工具函数、通用组件和样式变量，但页面、路由、布局和业务状态应保持清晰边界，不能互相混杂。

后台管理界面只按桌面端管理工作台设计和验收，不做手机端、移动端或小屏专用适配，不维护移动端菜单、移动端抽屉等状态。后台不是给移动端查看的页面，真实浏览器验收以桌面宽度为准。

如果后续公开前台展示界面需要移动端阅读体验，必须单独补充前台展示规格，不能把前台移动端规则反向套到后台管理界面。

## 后续业务设计原则

具体业务模块应在后端对应能力完成或进入开发阶段后，再单独补充前端业务规格。

每个业务模块的前端设计必须以对应后端接口文档为准，不能提前凭空设计字段、状态和交互流程。

如果后端接口尚未实现，前端可以预留模块位置，但不编写依赖假数据的完整业务页面。

## 工程边界

前端项目只负责：

- 页面展示。
- 用户交互。
- 表单输入。
- 路由跳转。
- 调用后端接口。
- 保存必要的前端状态。
- 展示后端返回的数据和错误信息。
- 对前端流转逻辑做测试验证。

前端项目不负责：

- 数据库表设计。
- SQL 查询。
- 后端业务校验。
- JWT 签发和验签。
- 密码加密。
- 后端权限拦截。
- 后端日志记录。
- 文件真实存储策略。
- 生产环境部署架构。

## agent skills 协作边界

UI 视觉、页面结构、布局细节、交互动效等内容，不在本前端工程规格中重复定义。

这些内容后续统一交给项目级 `.agents` 目录中的 agent skills 约束和生成。等 `.agents` 目录移动到 `D:\daimai\项目学习\` 后，前端项目和后端项目都按上级统一 agent 规则执行。

前端工程规格只约束：

- 目录归属。
- 路由组织。
- 请求封装。
- 接口类型。
- 状态管理。
- 本地缓存。
- 错误处理。
- 测试验证。
- 环境配置。
- 前后端联调方式。

agent skills 可以负责 UI 和页面组织的具体产出，但产出的代码必须接入本工程定义好的数据流。

## 交付边界

本项目的前端工程按 AI 交付型工程管理，优先保证可用、可维护、可验证。

用户不需要把精力放在手写复杂 UI、组件样式或前端生态细节上。视觉设计、复杂组件结构、样式细节和交互动效由 agent skills 辅助完成。

前端允许使用成熟工具提高交付质量，但新增工具必须服务真实工程问题，不能为了堆技术而引入。

新增技术必须满足两个条件：

- 能解决当前项目中的真实问题。
- 能用清晰文档说明它在工程中的职责和边界。

## 命名约定

前端项目目录名称暂定为：

```text
blog-web
```

该名称表达项目职责：个人博客前端工程，不限定为后台管理端。

如果后续需要拆分为多个独立前端应用，例如前台站点和后台管理端分开部署，应先补充新的工程拆分规格，再调整目录和构建方式。
