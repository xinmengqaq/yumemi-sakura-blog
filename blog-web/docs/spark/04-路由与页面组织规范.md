# 04-路由与页面组织规范

## 路由设计目标

路由用于组织前台页面、后台页面和错误页面。

本项目后续会同时承载博客前台展示和后台管理界面，因此路由设计必须提前区分访问区域，但不在当前阶段展开具体业务页面细节。

路由设计要满足：

- 前台页面和后台页面路径清晰分离。
- 需要登录的页面有统一守卫。
- 页面文件位置和路由路径容易对应。
- 后续新增业务页面时不破坏已有结构。

## 路由区域

路由分为三个区域：

```text
/              前台展示区域
/admin         后台管理区域
/error         错误页面区域
```

说明：

- `/` 用于博客前台展示。
- `/admin` 用于后台管理。
- `/error` 用于统一错误页面。
- 未匹配路径进入 404 页面。

`admin` 只表示后台管理路由前缀，不代表整个前端项目只服务后台。

## 页面目录对应关系

路由区域和 `views` 目录保持对应。

```text
src/views/
├─ front/       # 对应 /
├─ admin/       # 对应 /admin
└─ error/       # 对应 /error 和 404
```

示例：

```text
src/views/front/HomeView.tsx
src/views/admin/LoginView.tsx
src/views/admin/DashboardView.tsx
src/views/error/NotFoundView.tsx
```

## 布局对应关系

前台和后台使用不同布局。

```text
src/layout/
├─ FrontLayout.tsx
├─ AdminLayout.tsx
└─ BlankLayout.tsx
```

推荐关系：

- 前台展示页面使用 `FrontLayout`。
- 后台登录页使用 `BlankLayout`。
- 后台管理页使用 `AdminLayout`。
- 404 等错误页可以使用 `BlankLayout` 或独立错误布局。

布局只负责页面框架，不处理具体业务请求。

## 基础路由结构

基础路由结构建议如下：

```text
/
├─ 前台首页
├─ 前台其他公开页面
/admin/login
├─ 后台登录页
/admin
├─ 后台首页
├─ 后台其他管理页面
/error/*
├─ 错误页面
*
└─ 404 页面
```

当前阶段可以只保留必要入口和占位页面，不提前实现所有业务页面。

## 路由配置文件

路由配置放在：

```text
src/router/
├─ index.tsx
├─ routes.tsx
└─ guards.tsx
```

### index.tsx

负责创建路由实例，并导出给应用入口使用。

### routes.tsx

负责声明路由表。

路由表只描述：

- 路径。
- 页面组件。
- 布局关系。
- 是否需要登录。
- 页面标题等基础元信息。

路由表不写接口请求，不写表单逻辑，不写复杂状态处理。

### guards.tsx

负责路由守卫。

路由守卫只处理通用访问规则，例如：

- 未登录访问后台管理页面时跳转登录页。
- 已登录访问后台登录页时跳转后台首页。
- Token 无效时清理登录态并跳转登录页。
- 未匹配路由进入 404 页面。

## 路由元信息

路由可以定义少量元信息，用于守卫和页面标题。

示例字段：

```ts
type RouteMeta = {
  title?: string
  requiresAuth?: boolean
  guestOnly?: boolean
}
```

字段含义：

- `title`：页面标题。
- `requiresAuth`：是否需要登录。
- `guestOnly`：是否仅未登录用户访问，例如登录页。

元信息只放通用规则，不放业务权限细节。

## 登录态判断

后台管理页面需要登录态。

当前阶段登录态判断以 Token 为基础：

- 本地没有 Token：视为未登录。
- 本地有 Token：可以进入后台页面。
- 需要确认 Token 有效时，通过后端校验接口验证。
- 后端返回未登录或 Token 过期：清理本地登录态，跳转登录页。

前端不能自行解析 JWT 来判断权限或密码版本。JWT 的签发、验签和过期判断属于后端职责。

## 页面组织规范

页面文件放在 `views` 目录。

页面组件负责：

- 调用对应 `api` 方法或使用封装后的数据 Hook。
- 读取必要的前端状态。
- 把后端返回的数据展示出来。
- 显示加载中、空数据和错误状态。
- 处理用户触发的操作入口。

页面组件不负责：

- 创建 Axios 实例。
- 拼接完整后端地址。
- 直接解析 JWT。
- 实现后端权限判断。
- 伪造后端数据字段。
- 写复杂通用工具函数。
- 放大量可复用 UI 组件代码。

## 前台页面规范

前台页面面向访客访问，默认不需要登录。

前台页面重点是展示后端公开接口返回的数据。

前台页面可以包括：首页、文章列表、文章详情、分类文章、标签文章、说说列表、相册展示、友链展示、留言区域。

这些页面只在后端对应接口进入开发阶段后再展开具体设计。

## 后台页面规范

后台页面面向管理员访问，除登录页外默认需要登录。

后台页面重点是展示后端管理接口返回的数据，并提供管理操作入口。

后台页面可以包括：登录页、后台首页、个人资料页、文章管理、分类管理、标签管理、评论管理、留言管理、文件管理、站点配置。

这些页面只在后端对应接口进入开发阶段后再展开具体设计。

## 错误页面规范

错误页面放在 `views/error`。

至少应规划：

- 404：路径不存在。
- 403：无权限或禁止访问。
- 500：系统错误或前端无法恢复的异常。

当前阶段可以先实现 404，其他错误页面后续按需要补充。

## 页面接入边界

生成或新增路由页面时，必须先确认：

- 页面属于前台、后台还是错误区域。
- 页面是否需要登录态。
- 页面应使用哪个布局。
- 页面依赖哪个后端接口。
- 接口请求是否已经封装在 `api` 目录。
- 接口类型是否已经定义在 `types` 目录。

不能为了页面效果临时创建一套独立路由结构。

不能把前台页面放进后台目录，也不能把后台管理页面放进前台目录。

## 占位页面规则

如果后端接口尚未完成，前端可以保留占位页面。

占位页面只能说明该模块暂未接入真实接口，不能伪造完整业务数据。

占位页面不作为正式功能验收依据。
