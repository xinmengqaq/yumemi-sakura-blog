# 文章运营工作台分类标签 UI 设计规格

## 设计背景

本规格用于把后端第五至第七阶段已经完成的能力接入现有博客后台文章管理，包括文章分类、文章标签、多表关联、状态流转、置顶推荐、批量删除以及组合筛选。

本次采用增量扩展方案，不重做已经完成的文章列表和块状 Markdown 编辑器。视觉与交互以当前真实页面为准：

- 文章列表基线：`output/playwright/task19-articles-initial.png`。
- 编辑页基线：`output/playwright/task19-shortcut-drawer.png`。
- 后台继续使用冷中性工作台、赤陶红强调色、细边框、低圆角和无页面阴影。
- `docs/前端设计目录/文章UI/imagegen/` 下的蓝色旧原型不作为本次视觉依据。

## 设计目标

- 保留现有文章管理工作流，在原页面中补齐第五至第七阶段能力。
- 让管理员能够维护文章分类和文章标签，并在编辑文章时完成关联。
- 提供高效但克制的状态流转、置顶、推荐和批量删除操作。
- 所有界面能力严格服从真实后端契约，不提供后端尚未支持的搜索、排序或批量更新能力。
- 保持正文编辑器独立边界，不因文章业务字段扩展而修改编辑器内部模型。

## 后端契约范围

### 文章

文章保存字段增加：

```ts
type ArticleSaveParams = {
  title: string
  summary?: string
  content: string
  coverUrl?: string
  categoryId?: number | null
  tagIds: number[]
  status: 'draft' | 'published' | 'hidden'
  isTop: boolean
  isRecommend: boolean
}
```

文章分页支持 `keyword`、`status`、`categoryId`、`tagId`。排序由后端固定处理，不在前端提供排序选项。

文章分页列表不返回、不展示标签集合。`tagId` 只用于列表筛选；文章标签关联由 `ArticleDetailVO` 返回，供编辑页回填和保存使用。

文章还提供单独的状态、置顶、推荐更新接口和批量删除接口。批量删除返回实际删除数量。

### 分类

分类包含名称、描述、排序值、显示状态、关联文章数、创建时间和更新时间。

- 名称必填，最多 50 个字符。
- 描述可空，最多 200 个字符。
- 状态为 `visible` 或 `hidden`。
- 分类列表只支持状态筛选，不支持关键词搜索和分页。
- 分类被文章占用时不允许删除。

### 标签

标签包含名称、关联文章数、创建时间和更新时间。

- 名称必填，最多 50 个字符。
- 标签列表支持名称关键词筛选，不支持分页。
- 标签被文章占用时不允许删除。

## 导航与路由

侧边栏“文章管理”升级为可展开父级导航：

```text
文章管理
├─ 文章分类管理
└─ 文章标签管理
```

父级文字点击进入现有文章列表，右侧箭头独立控制展开和收起。

| 页面         | 路由                         |
| ------------ | ---------------------------- |
| 文章列表     | `/admin/articles`            |
| 新建文章     | `/admin/articles/new`        |
| 编辑文章     | `/admin/articles/:id/edit`   |
| 文章分类管理 | `/admin/articles/categories` |
| 文章标签管理 | `/admin/articles/tags`       |

选中规则：

- 文章列表、新建页和编辑页只选中“文章管理”父级。
- 分类页选中“文章分类管理”，标签页选中“文章标签管理”，父级同时保持所属模块状态。
- 进入分类或标签路由时自动展开文章导航。
- 顶部栏在分类和标签页分别显示完整页面名称；新建和编辑页顶部栏继续显示“文章管理”。

## 文章列表增量设计

### 筛选区

默认行保留现有关键词、状态、查询和重置。新增“更多筛选”按钮，展开后显示：

- 文章分类：单选，可清空。
- 文章标签：单选，可清空。

高级筛选收起后仍显示已应用条件摘要。后端没有排序参数，因此前端不提供“最近更新”等排序选项，列表遵循后端固定的置顶优先和发布时间倒序规则。

### 文章列表项

保留现有封面、标题、摘要、阅读量、评论数、发布时间和更新时间。增量显示：

- 发布状态。
- 启用时显示“置顶”和“推荐”标识。
- 分类名称。

默认操作区调整为“编辑”和更多菜单。更多菜单根据当前状态动态提供：

- 发布、转为草稿或隐藏。
- 置顶或取消置顶。
- 推荐或取消推荐。
- 删除。

删除放在菜单末尾危险区。批量模式下隐藏单篇更多菜单。

### 批量管理

列表默认不常驻复选框。点击“批量管理”后进入批量模式：

- 每条文章显示复选框，并支持当前页全选。
- 显示已选数量、批量删除和退出批量管理。
- 翻页前清空当前选择，不保留跨页隐形选择。
- 删除确认显示选择数量和部分文章标题。
- 成功后显示后端返回的实际删除数量。

## 文章编辑页右侧设计

正文编辑器和所有编辑器浮层保持现状。本次只扩展当前右侧“文章信息”栏。

编辑页顶部使用单行工作区标题栏：返回文章列表、页面标题和保存状态位于同一行。正文编辑器直接进入内容区，不重复展示“正文”标题和用途说明。

右栏顺序：

```text
文章信息
标题
摘要
封面 URL
封面预览

内容关联
文章分类
文章标签

发布设置
状态
置顶文章
推荐文章

保存文章
```

各组之间只使用小标题和细分隔线，不嵌套新卡片。

### 文章分类选择

- 使用可搜索单选控件，允许清空。
- 新建文章只显示可见分类。
- 编辑文章若已关联隐藏分类，仍显示该分类并标注“已隐藏”，但隐藏分类不能被新选中。
- 无匹配结果时提供“新建文章分类”。
- 快捷新建只填写名称，创建后自动选中；描述为空、排序值为 `0`、状态为显示。

### 文章标签选择

- 使用可搜索多选控件。
- 已选标签以紧凑标签项展示，可单独移除。
- 无匹配结果时提供“新建文章标签”。
- 快捷创建成功后自动加入已选列表。
- 标签过多时限制控件高度并在控件内部滚动或折叠，避免右栏无限增长。

快捷创建不会自动保存文章，成功后文章仍显示“有未保存修改”。

### 发布设置

- 状态继续使用草稿、已发布、隐藏单选组。
- 置顶和推荐使用两个紧凑开关，并配一行结果说明。
- 新建文章默认草稿、未置顶、未推荐。
- 编辑页保存时统一提交分类、标签、状态、置顶和推荐字段。
- 列表快捷操作使用后端独立 `PATCH` 接口，不要求进入编辑页。

## 文章分类管理页

页面使用标题、说明、新建按钮、状态筛选和连续行管理列表。列表采用单一外框、横向分隔、无竖线和低对比表头，不使用旧版蓝色表格或独立卡片堆叠。

列表字段：

- 分类名称与描述摘要。
- 显示或隐藏状态。
- 关联文章数量。
- 排序值。
- 更新时间。
- 编辑与更多菜单。

新建和编辑使用小型对话框，字段包括名称、描述、排序值和状态。排序使用数字步进输入；后端没有批量重排接口，因此不提供拖拽排序。

分类列表只提供后端已有的状态筛选，不制作无效的关键词搜索框。

## 文章标签管理页

页面骨架与分类管理一致，列表字段包括：

- 标签名称。
- 关联文章数量。
- 更新时间。
- 编辑与更多菜单。

支持后端已有的标签名称关键词筛选。新建和编辑使用小型对话框，字段只有标签名称。

## 删除与占用反馈

- 分类和标签的删除入口保持可发现。
- `articleCount > 0` 时，点击删除直接解释当前关联文章数量，并提供前往文章列表按该分类或标签筛选的入口。
- `articleCount = 0` 时显示最终删除确认。
- 页面数量可能过期，最终仍以后端删除结果为准；失败时显示后端 `msg`。
- 删除成功后刷新管理列表、文章筛选选项和编辑页关联选项缓存。

## 数据流与缓存

继续使用 `types -> api -> queries -> views` 边界，页面禁止直接调用 Axios。

- 文章列表并行获取分类、标签选项和文章分页数据。
- 编辑页并行获取文章详情、分类列表和标签列表。
- 快捷新建成功后更新对应选项缓存并写入当前表单。
- 管理页新增、修改、删除后刷新对应 Query Cache。
- 单篇状态、置顶或推荐更新成功后刷新当前文章列表。
- 保存文章成功后更新文章详情缓存并刷新文章列表。
- 更新接口响应可能不包含完整标签集合；保存成功后，响应缺失的分类、标签、置顶和推荐字段沿用本次提交值，不能把已保存关联在本地表单中清空。
- 关联数据加载失败时允许继续编辑正文，但阻止保存，避免把原有关联误提交为空。

## 错误处理

- 重名、占用、非法状态和参数错误统一展示后端 `msg`。
- 快捷创建失败时保持选择浮层和当前输入。
- 单篇状态操作采用请求成功后再更新界面的保守方式。
- 批量删除期间锁定批量操作栏，防止重复提交。
- 文章保存失败后保留正文、分类、标签和发布设置，不重新加载覆盖用户输入。
- 隐藏分类不会从已有文章表单中被强制清除。

## 测试与验收

自动化测试只覆盖非 UI 层：

- 分类、标签和文章 API 的路径、参数与请求体契约。
- TanStack Query 的查询键、缓存失效和详情缓存更新。
- 后端 Service、Mapper、分页、关联数据与事务行为。

不新增或扩写 Testing Library 页面、导航、组件和视觉测试。UI 行为、布局和交互统一通过真实浏览器连接真实后端验收。

真实浏览器验收覆盖：

- 最新文章列表和编辑页视觉基线的一致性。
- 更多筛选展开前后的布局稳定性。
- 默认列表与批量模式切换时不发生明显横向跳动。
- 更多菜单和选择浮层不被容器裁切。
- 右栏增加关联与发布设置后的滚动、换行和长标签表现。
- 分类、标签对话框的校验、占用提示和长名称表现。
- 快捷创建、加载、空状态、无结果、请求失败和保存失败。
- 正文编辑器现有块工具、文字工具、表格工具和快捷键浮层不受影响。

正式实现交付前至少通过 `npm run typecheck`、精确 ESLint、API 与 Query 相关 Vitest、`npm run test:run`、`npm run build` 和后端测试，并使用真实后端验证文章关联保存、组合筛选、状态流转和批量删除闭环。

## 非目标

本次不实现：

- 分类层级或树形分类。
- 标签层级。
- 分类拖拽排序。
- 分类关键词搜索。
- 分类和标签分页。
- 批量修改文章状态、置顶、推荐、分类或标签。
- 跨页批量选择。
- 后端没有提供的自定义排序方式。
- 修改块状 Markdown 编辑器的业务无关能力。

## 实施确认

- 功能必须连接真实 Spring Boot 后端完成闭环，不使用假数据、假接口或前端模拟成功。
- UI 不新增自动化测试；页面、导航、组件、布局和交互通过真实桌面浏览器验收，并以实际浏览器窗口尺寸记录结果。
- 保持 `types -> api -> queries -> views` 边界，页面只编排状态；分类标签领域、文章关联字段和批量操作按现有实施计划拆分到聚焦文件。
- 优先复用现有 `Button`、`Input`、`FormField`、`Modal`、`ConfirmDialog`、`Alert`、CSS 变量和原生表单控件，不另建通用 CRUD 框架或第二套弹窗系统。
- 默认不安装 GSAP；菜单、更多筛选和子导航使用约 `120-180ms` 的 CSS `transition`，并遵守 `prefers-reduced-motion`。只有真实浏览器证明 CSS 无法维护复杂序列时，才单独评估动画库。

## UI 规范锁定

本次属于保留式后台产品 UI 扩展。`taste-skill` 只用于校准可访问性、状态完整性、颜色和形状一致性，不把营销页模式套入后台工作台。

- 设计参数：`DESIGN_VARIANCE 6`、`MOTION_INTENSITY 4`、`VISUAL_DENSITY 5`。布局允许适度错落，但操作区保持可扫描；动画只解释展开、关闭和状态切换。
- 继续使用现有冷中性色和赤陶红单强调色，不增加紫蓝渐变、发光、玻璃拟态或跨区反色。
- 继续使用现有 4-6px 低圆角体系。按钮、输入、菜单、弹窗和连续行列表不得各自形成不同圆角语言。
- 管理页采用标题、筛选工具条和单一连续行容器，不使用卡片套卡片、等宽三卡或装饰性状态圆点。
- 所有按钮文字在桌面端保持单行；输入标签位于控件上方；焦点、错误、加载、空状态和占用状态必须完整可见。
- 菜单支持 Escape、外部点击关闭和键盘焦点；开关基于原生 checkbox；所有显隐过渡在减少动态效果设置下立即完成。
- 页面可见文案不使用破折号作为装饰，不增加版本号、步骤编号、滚动提示或无业务含义的微标签。
- 后台本轮只按真实桌面窗口验收，不新增移动端布局目标；现有 `min-width: 1080px` 工作台约束保持不变。

## 实现结果与验收记录

已完成文章父子导航、共用操作菜单与原生开关，并接通分类管理、标签管理、文章组合筛选、单篇状态操作、当前页批量删除、编辑页分类标签关联、快捷创建和完整发布设置。

真实浏览器使用 Playwright CLI 的 Chromium，会话最终实际视口为 `1920 x 945`，设备像素比为 `1`。验收只记录实际窗口事实，不把视口尺寸写入业务 CSS。

真实后端验收使用管理员 `xinmengqaq` 完成登录，已确认：

- 分类与标签的新建、编辑、筛选和删除均连接真实接口；占用项准确提示关联文章数量，“查看关联文章”分别生成 `categoryId` 和 `tagId` 筛选 URL。
- 新建文章通过编辑页快捷创建分类和标签，保存后详情正确回填分类、多个标签、草稿状态、置顶和推荐字段。
- 文章列表组合使用已发布状态、分类和标签时只返回目标文章；发布、取消置顶和取消推荐均通过独立 `PATCH` 接口即时更新。
