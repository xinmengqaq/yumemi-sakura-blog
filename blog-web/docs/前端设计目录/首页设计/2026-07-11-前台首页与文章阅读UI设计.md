# 前台首页与文章阅读 UI 设计

## 1. 文档目的

本文定义个人博客前台首期 UI 与数据契约，覆盖首页、文章列表和文章详情阅读页，为后续 Product Design 原型生成、前端实现和后端公开接口开发提供统一依据。

本文只描述已经确认的首期范围。视觉原型用于确定布局与风格，不能替代接口数据，也不能据此提前扩展未确认功能。

实施基线（2026-07-12）：已依据 Spring Boot `PublicContentController` 及公开 VO 复核本文件的接口契约。原型文件的实际画布尺寸为首页 `863×1822`、文章列表 `1023×1537`、文章详情 `1024×1536`；这些尺寸只用于确认信息层级和内容节奏，前端仍须按真实浏览器视口重新计算容器、栏宽和留白。

视觉识别增补（2026-07-13）：已确认“梦见樱町”正式站名、寒蝉圆黑字体体系、樱花轨道 Logo、Pixel2Motion 小体量品牌揭示、首页分区滚动入场、全前台日系动画 UI 图标、环形轨道页面过渡、桌面花瓣指针效果，以及 `gpt-image-2` 生成与 BiRefNet-General 透明背景处理规范。浅灰媒体占位只表示资源缺失，不属于主题依据。

## 2. 产品目标

前台面向公开访客，首期目标是：

- 通过有辨识度的二次元首页建立个人博客气质。
- 让访客发现精选文章、浏览最新文章并进入阅读。
- 提供适合内容发现的公开文章筛选和分页能力。
- 提供舒适、稳定的文章正文阅读体验。
- 接入真实分类、标签、浏览、评论统计、点赞和精选状态。
- 通过樱花车站氛围组件、全屏花瓣和克制的页面过渡强化二次元辨识度，同时不妨碍内容阅读。

## 3. 术语

- 后端字段 `isRecommend` 在前台统一展示为“精选”，前台文案不得出现“推荐文章”。
- “评论”在本期只指后端返回的评论数量统计，不包含评论交互模块。
- 点击“文章”导航主入口表示浏览全部文章；其子菜单只包含后端返回的公开分类。
- “占位图片”指接口没有有效图片或图片加载失败时显示的中性浅灰纯色区域，不是固定业务内容。

## 4. 首期范围

### 4.1 页面与路由

| 页面     | 路由            | 职责                                 |
| -------- | --------------- | ------------------------------------ |
| 首页     | `/`             | 展示头部场景、精选内容和少量最新文章 |
| 文章列表 | `/articles`     | 展示全部公开文章，支持筛选与分页     |
| 文章详情 | `/articles/:id` | 展示单篇公开文章及点赞入口           |

三个页面均使用 `FrontLayout`，默认公开访问，不依赖管理员登录态。

### 4.2 导航

导航栏首期只包含：

- 首页：跳转 `/`。
- 文章：主入口跳转不带 `categoryId` 的 `/articles`，表示全部文章；同时提供分类子菜单。

文章分类子菜单不重复提供“全部文章”。子菜单只展示公开分类接口返回的真实分类，并按后端顺序排列；选择分类后进入 `/articles?categoryId=<id>`。访客点击“文章”主入口时回到全部文章并清除已有分类条件。

分类选择只存在于导航栏“文章”的子菜单中。文章列表页不得在筛选区或结果区重复展示“全部文章”及各分类的横向标签、按钮组、下拉框或其他分类选择器。列表页只读取 URL 中的 `categoryId` 作为当前分类条件，并在当前条件摘要中展示和移除该条件。

桌面端允许悬停和点击展开。移动端使用点击折叠，选择分类后自动收起导航。

桌面导航使用独立的全宽头部轴线，不受正文居中容器宽度限制；Logo 位于左侧安全边距内，动画开关位于右侧。宽桌面上主菜单左边缘与首屏正文容器左边缘对齐；空间不足时回到视口居中，避免与 Logo 重叠。移动端继续使用 `16px` 安全边距，并保持左侧 Logo、右侧汉堡按钮。

分类子菜单使用樱粉纸张浮层与柔和悬停态，不使用后台式深色面板；动画开关和首页站牌使用相同的前台氛围按钮语言。

## 5. 视觉方向

### 5.1 整体风格

- 二次元、温柔、轻盈、具有个人刊物气质。
- 主体底色使用暖白到极浅樱粉再回到暖白的低对比纵向渐变，樱花粉用于柔和渐变、分隔、选中态和少量花瓣装饰；背景不得出现明显色块或装饰光斑。
- 可用少量晨空蓝、清新绿和深樱紫文字平衡粉白配色。
- 不使用科技风、赛博风、霓虹、深色模式、玻璃拟态或企业后台视觉。
- 前台浮层与氛围按钮统一使用暖白樱粉纸张质感、粉色描边和轻柔粉色阴影；禁止复用后台式深灰矩形菜单或按钮。
- 不将所有内容制作成同尺寸卡片，不允许卡片嵌套卡片。
- 前台界面文字统一使用寒蝉圆黑体，不再使用 Georgia、系统宋体或手写体；文章正文只继承基础阅读字体，不覆盖后台编辑器产生的作者格式。
- 卡片与图片圆角不超过 `8px`，阴影少量且轻。

### 5.2 视觉基准生成

Product Design 阶段生成三张页面原型：

1. 先使用纯文本生成首页母版。
2. 再以首页母版作为图生图参考生成文章列表页。
3. 再以首页母版作为图生图参考生成文章详情页。

后两张必须继承首页的导航、字体、配色、樱花车站意象、波浪衔接、间距和组件语言。图生图只用于保持视觉一致，不允许照搬其他网站或未经授权的角色与图片。

### 5.3 页面内容骨架

- 所有页面的头部背景可以铺满视口宽度。
- 首页正文使用“左侧空白外栏 / 中央内容列 / 右侧空白外栏”的隐形三列骨架。在 `1440px` 宽桌面原型中，中央内容列建议控制在约 `760px` 至 `800px`，更宽视口优先增加两侧留白，不放大正文行长。
- 文章列表可以在中央区域内适度加宽，以容纳左右交错的封面和文字，但筛选器、列表和分页必须属于同一视觉轴线。
- 文章详情使用双栏阅读区，正文列约 `720px`，右侧阅读工具列约 `280px`，中间保留约 `64px` 间距。正文因此相对页面中心轻微左移，但整个双栏区域仍在视口中稳定居中。
- 页面外栏不是可见卡片或业务侧边栏，不绘制栏边框、底色和无关标题。
- 首期不为 Live2D 绘制轮廓、标记或专属占位，也不为了未来模型挤压、偏移当前内容。以后新增 Live2D 时必须重新确认布局。

### 5.4 动效与氛围系统

- 三个页面共享统一的车站氛围动效语言，动画用于表达“抵达、轨道和春风”，不能变成持续抢夺注意力的展示效果。
- 全站使用固定覆盖整个视口的樱花花瓣动画层。动画层必须设置 `pointer-events: none`，不得阻断点击、滚动、文本选择或辅助技术访问。
- 头部场景保持正常花瓣密度；进入正文核心区域的花瓣自动降低到约 `8%` 至 `15%` 透明度，并以小尺寸为主；经过页面外侧留白时恢复正常可见度。花瓣不得停留在标题、代码块、筛选控件或按钮上。
- 桌面端同时显示约 `12` 至 `18` 片花瓣，移动端降低到约 `5` 至 `8` 片。花瓣缓慢斜向飘落并偶尔轻微回旋，不使用高速、暴风式或大面积遮挡动画。
- 花瓣使用固定数量的 Canvas 粒子复用，不为每片花瓣持续创建 DOM 节点。页面失焦或不可见时暂停动画。
- 提供小型樱花图标开关。开关只保存当前设备的视觉偏好，不属于用户资产，不依赖登录或后端。
- 页面切换必须有统一过渡：旧页面短暂淡出后显示暖白樱粉半透明遮罩，中心使用简化樱花轨道 Logo，外圈为环形轨道加载器。浅樱粉底轨上有一段樱莓红进度弧顺时针移动，前端花瓣经过四个站点时依次点亮；目标页面就绪后完成收尾并淡出，新页面以 `12px` 至 `20px` 的轻微纵向位移和透明度变化进入。常规跳转总时长控制在约 `450ms` 至 `700ms`，不得使用整屏横滑、长时间遮罩或为展示动画故意延迟导航。
- 页面尚未就绪时圆环可以持续匀速运行，但数据请求、路由解析和动画状态必须解耦；加载结束由真实就绪状态触发，不使用伪进度数字。`prefers-reduced-motion` 命中时取消旋转和站点逐个点亮，仅显示静态 Logo 与圆环状态色变化。
- 首页头图在标题附近加入小体量樱花轨道 Logo reveal。图形标宽度约为主标题宽度的 `18%` 至 `24%`，放在标题上方或左上安全区，不占据头图背景主体、不遮挡场景焦点。动画依次完成轨道描线、站点点亮、五瓣樱花展开和 HTML 标题淡入，结束后严格落在静态 Logo 最终态。
- 品牌完整开场在每个浏览器会话内只自动播放一次，使用带动画版本号的会话状态控制；同一会话返回首页直接显示最终态。桌面精确指针悬停 Logo 时只重播轻量花瓣绽放，不重播完整开场；触屏不模拟悬停。会话存储不可用时直接显示静态最终态。
- “本期精选”进入视口时播放一次“樱花站台抵达”：章节轨道线先延伸，樱花站点章点亮并轻微回弹，精选文章再按阅读顺序从轨道方向移动约 `16px` 至 `24px` 并由透明变为清晰，最后以一次短促轨道高光收尾。“最近抵达”各文章从下方约 `20px` 依次淡入，项目间隔约 `70ms` 至 `100ms`。
- 首页滚动入场只负责揭示已经加载并存在于 DOM 的文章，不把滚动位置作为数据请求条件。使用 ScrollTrigger 的一次性离散触发，不固定页面、不使用 `scrub` 绑定滚动进度、不在回滚时反向重播；React 组件通过有作用域的 `useGSAP` 建立并在卸载时清理，字体或图片改变布局后刷新触发位置。
- 桌面精确指针设备使用“花瓣伴随”鼠标效果：一片 `10px` 至 `14px` 的单花瓣在系统指针旁轻量跟随并按移动方向改变角度，悬停可交互元素时绽放为五瓣樱花，点击时散出 `3` 至 `5` 片小花瓣并在约 `400ms` 内消失。系统指针本身、文本光标、拖动和缩放光标不得被替换；输入、选择文本和拖动期间暂停装饰，触屏设备和 `prefers-reduced-motion` 环境完全关闭该效果。
- 简单悬停使用 CSS 过渡；滚动轨道、章节联动和页面进入序列使用 GSAP。优先动画 `transform` 和透明度，避免用 `top`、`left`、`width` 或 `height` 制造位移。
- 必须支持 `prefers-reduced-motion`。命中时停止花瓣飘落、视差和滚动入场，只保留即时状态变化与少量静态花瓣装饰。

### 5.5 字体与文字效果系统

前台首页、文章列表、文章详情和共享导航统一自托管寒蝉圆黑体。只加载实际使用的 Regular、Medium、Bold 和 Heavy 字重，`Noto Sans SC` 仅作为字体加载失败时的回退；不得依赖访客操作系统中的中文字体产生最终效果。字体加载使用 `font-display: swap`，首屏关键字重需要预加载，其他字重按页面实际使用范围加载。

文字色彩以正式樱花车站场景为依据，不得从浅灰占位图反推主题：

- 深樱紫 `#4a3042`：正文入口标题、导航深色态和主要文字。
- 樱莓红 `#c95778`：标题偏移阴影、悬停和重点反馈。
- 花瓣粉 `#f19ab2`：描边、图标、轨道线和选中态。
- 浅樱粉 `#ffd9e4`：辅助文字和柔和状态。
- 暖白 `#fff9fc`：头图主文字和浅色表面。
- 春叶绿 `#6f9c78`：少量状态或统计点缀，不与樱粉争夺主视觉。

不同位置的文字表现如下：

- 顶部博客名使用 Bold 暖白字与樱粉品牌标；导航使用 Medium，当前项显示樱粉轨道线。
- 头图辅助文案使用 Medium 暖白或浅樱粉；欢迎语使用 Regular 暖白和克制的半透明樱紫阴影。
- 首页主标题“梦见樱町”使用 Heavy 动画标题牌效果，是全页唯一允许同时使用填充、描边和偏移阴影的强视觉文字。
- 站台时钟使用 Bold 和等宽数字特性；问候与站牌短句使用 Medium，不改造成电子屏字体。
- “本期精选”“最近抵达”等章节标题使用 Bold 深樱紫，配自定义樱花站点章；文章入口标题使用 Bold，悬停转为樱莓红并轻微上移。
- 摘要使用 Regular 暖紫灰，元数据使用 Regular 淡紫灰；不添加描边、渐变或持续动画。
- 字号在明确断点间切换，不使用随视口宽度连续缩放；文字字距保持 `0`，不依赖拉大字距制造日系感。

文章详情正文不属于主题文字特效范围。正文只提供寒蝉圆黑 Regular 的基础字号、行高、段落间距和阅读宽度；后台编辑器产生的颜色、加粗、高亮、标题、引用和其他受支持格式优先，前台不得用全局颜色、描边、阴影、渐变、悬停或入场效果覆盖作者表达。代码块继续使用独立等宽字体。

### 5.6 品牌 Logo 与全前台图标系统

前台个人站正式名称为“梦见樱町”。“梦见”保留个人网名中最具辨识度的“梦”，表达让想象和创作被看见；“樱町”把站点定义为明亮、温暖、持续生长的私人樱花小镇。“薪”不直接进入站名，也不使用夜火意象，而通过春日新生、持续创作和不断建设小镇形成暗喻。原“春日轨迹”只作为早期临时名称，不再出现在正式界面、浏览器标题、favicon 名称或品牌资产说明中。

品牌 Logo 采用“樱花轨道标”：五瓣樱花与两条弧形铁轨融合，中心保留简化站点圆点。Logo 图形与“梦见樱町”文字分离，图像模型只生成无文字图形，品牌文字由寒蝉圆黑 Bold 实时渲染，避免生成错字和缩放模糊。桌面端显示图形标与品牌文字，极窄移动端允许只显示简化图形标；简化标同时用于 favicon、页面切换加载器中心和首页小体量品牌揭示。

首页、文章列表和文章详情统一重做图标语言，覆盖导航、内容发现、文章元数据、阅读工具、操作和氛围状态。图标可以重新设计，但搜索仍保持放大镜、关闭仍保持叉号、点赞仍保持心形等基础语义，不能为了原创牺牲识别性和可访问名称。

- 图标采用日系动画 UI 风格：干净二维色块、细樱紫轮廓、少量暖白高光和透明背景，不使用水彩、玻璃拟态或 3D 拟物。
- 基准尺寸为 `24px`，紧凑元数据使用 `16px`，重点氛围图标使用 `32px`；每个小图标最多使用两种颜色。
- 不给所有图标套统一方形底板。只有当前、悬停或需要扩大点击区域时出现浅樱粉状态底，点击目标仍满足可访问尺寸。
- 功能图标保留熟悉轮廓并按同一端点、线宽、基线和樱花细节规范化；Logo、列车、站点章、车票、信号灯和状态插图允许使用完整原创造型。
- 点赞由樱紫轮廓过渡为樱莓红填充并散出两片花瓣；复制成功由车票纸张轮廓切换为樱花站戳；搜索激活时放大镜内出现浅花瓣；筛选生效时调节线旁亮起樱粉站点；翻页使用方向明确的轨道箭头。
- 所有非文字图标必须有可访问名称或与相邻可见文字形成明确语义；纯装饰图形必须从辅助技术中隐藏。

首批品牌资产包括樱花轨道主标、简化轨道花标、单片动态花瓣、五瓣绽放图标、樱花站点章、动画开关盛开/静止双状态、空车票状态图和暂停车信号状态图。圆环加载器、站点点亮和基础状态变化由代码绘制，不把整段动画烘焙成图片。

### 5.7 生成式资产与透明背景处理

Logo、主题图标和状态插图允许使用 `gpt-image-2` 的 Garden Mode A Python 脚本生成。先生成统一风格概念板确认造型，再为每个实际资产单独生成透明高分辨率 PNG；不得直接裁切概念板网格作为正式图标，也不得让图像模型生成品牌中文文字。

透明背景处理固定使用 MIT 许可的 `rembg` 与 BiRefNet-General 模型：

1. 生成时优先请求透明背景 PNG，并保留原始响应和提示词。
2. 检查 Alpha 通道、主体边界和背景残留；不合格时使用 `rembg + birefnet-general` 重新分割并启用 Alpha Matting。
3. 清理低透明度杂点，保留约 `10%` 安全留白，不得吃掉花瓣尖端、轨道细线和 Logo 内部负形。
4. 分别在暖白、深樱紫、正式樱花头图和透明棋盘格上以 `400%` 放大检查白边、黑边、锯齿、断线和半透明脏像素。
5. 连续两次仍无法得到干净边缘的资产必须重新生成，不能用手工涂抹掩盖结构错误。

源文件保留无损 PNG；网页交付版本按实际透明度和尺寸输出压缩 WebP/PNG，并记录来源提示词、生成时间、模型和抠图模型。不得使用仅允许非商业自托管的背景移除权重。

樱花轨道 Logo 在透明位图验收后使用 GitHub `nolangz/pixel2motion` 提交 `3824573d4e522c244d82b810fc5b85c7858f6b66` 离线重建和编排动画。该提交采用经核验的 MIT 许可，不作为浏览器运行时依赖；仓库记录上游地址、提交号、许可、输入输出摘要和执行命令。矢量化与动画遵守：

1. 五瓣樱花、两条弧形轨道、站点圆点和必要高光分别成为带稳定 `id` 的语义部件，不使用结构选择器定位动画目标。
2. 采用能准确解释图形的最低复杂度平滑几何；自动追踪只作为测量起点，普通曲线不得交付像素阶梯、抖动轮廓或大量无意义节点。
3. 通过源图叠加、轮廓平滑审计和关键尺寸缩放检查确认静态 SVG；IoU 仅作为诊断，不能替代对花瓣尖端、轨道宽度、负形和站点位置的视觉判断。
4. 动画编排包含轨道描线、站点点亮、樱花展开和轻量悬停绽放，最终帧必须与已验收静态 SVG 在几何、颜色、尺寸和位置上一致。
5. Pixel2Motion 的独立展示 HTML、回放按钮、慢速开关、速度滑块和 QA 面板只用于制作与验收，不嵌入正式 React 页面。正式运行时只消费静态 SVG、必要动画样式和减少动态效果降级。
6. 交付 `logo.svg`、动画样式、motion spec、静态拟合叠加图、关键动画帧、最终帧对比和路径审计报告；SVG 或动画样式失败时仍显示静态图形标与正常 HTML 标题。

## 6. 首页设计

### 6.1 设计概念

首页采用“樱花轨道叙事”，不是常见的头图加文章卡片列表。车站、站牌、轨道和抵达被用作轻量视觉语义，页面仍以内容可读性为第一优先级。

### 6.2 首屏头部

- 使用全幅樱花车站场景作为最终视觉方向。
- 导航位于头部上方，保证文字对比度。
- 博客名称和一句欢迎语放在画面留白区域，不放入营销卡片。
- 主标题“梦见樱町”使用动画标题牌：Heavy 暖白填充、花瓣粉外描边和樱莓红短距离偏移阴影，不使用深蓝灰、霓虹发光或渐变文字。描边和阴影必须在桌面与移动端分别校准，不能压住字腔或降低中文识别；移动端减少描边和偏移量，不按视口连续缩放字号。
- 樱花轨道图形标位于主标题上方或左上安全区，宽度约为标题的 `18%` 至 `24%`；图形揭示和标题淡入属于同一开场时间线，但 SVG 不承载中文文字。动画不能扩展成覆盖头图大面积背景的场景演出。
- 头图下方使用约 `30px` 至 `50px` 高的单层低幅白色波浪衔接正文。
- 波浪保持浅、柔和，使用两层低幅错相位动态波浪，不形成大面积空白。
- 首屏底部应露出下一段内容，提示页面可以继续浏览。
- 头部提供轻量站台时钟，根据访客设备时间显示当前时刻和早晨、午后或夜间问候，不调用天气或其他外部接口。
- 站台时钟采用无容器排版，以粉色时钟图标、寒蝉圆黑 Bold 暖白等宽数字和轻量问候强化二次元车站语义；不使用手机系统组件式矩形，也不添加背景、边框或阴影卡片。
- 桌面端站牌与站台时钟在同一内容行内横向排列，不能贴在底部波浪边缘；二者均直接融入头图，不使用可见卡片容器。移动端空间不足时改为纵向排列。
- 站牌允许点击切换一条内置短句；短句是固定界面氛围文案，不保存访客记录。
- 同一浏览器会话首次进入首页时，远景列车可以缓慢经过一次。离开后返回首页不重复播放，避免持续打扰。

正式头图资产未完成时，使用同尺寸浅灰纯色占位，仍保留文字安全区和低波浪边界。

### 6.3 本期精选

- 本区所有内容必须限制在中央内容列内，两侧外栏持续留空。
- 左侧展示一篇大幅精选主文章，使用大封面、标题和摘要形成杂志封面感。
- 右侧沿细竖向轨道排列两至三张等高的次精选文章卡片，每张使用稳定比例的小幅封面、标题和元数据；轨道只负责串联卡片，不使用目录式编号。
- 精选内容由后端返回，不由前端根据浏览数或点赞数推断。
- 精选不足时按实际数量自然排版，不生成假文章补位。
- 移动端主精选后只保留前两张横向等高的次精选卡片，短轨道继续串联卡片并减少组件密度；最新文章区仍独立显示，不能因精选存在而隐藏。
- 精选为空时隐藏本区，让最新文章区自然上移。
- 精选区进入视口时播放一次“樱花站台抵达”：章节标题下的轨道线先延伸，樱花站点章点亮并轻微回弹，主精选与次精选按阅读顺序从各自轨道方向移动约 `16px` 至 `24px` 并淡入，最后以一次短促轨道高光收尾。动画不使用目录式编号，不随反复滚动重播。

### 6.4 最近抵达

- 本区完整收在中央内容列，不得为了错落效果把封面或文字延伸到页面外栏。
- 使用错落横向图文带展示少量最新文章。
- 不使用与文章列表页相同的连续列表，也不使用规则等宽宫格。
- 每项可以展示标题、摘要、发布时间及浏览、评论、点赞数量。
- 区域末尾提供“查看全部文章”入口，跳转 `/articles`。
- 图文带进入视口时从下方约 `20px` 依次淡入，项目间隔约 `70ms` 至 `100ms`；只动画 `transform` 和透明度，不改变最终排版尺寸，不因快速滚动延迟内容可见。

首页不展示筛选器、分页、分类墙或标签云。

## 7. 文章列表页设计

### 7.1 页面头部

- 延续统一导航。
- 使用与首页相同的樱花车站头图媒体层和遮罩，但保持较矮横幅与“文章”标题，不重复首页大型首屏。
- 列表横幅与首页复用同一动态双层波浪组件，不允许退化为静态弧形。
- 正式横幅未完成时使用对应比例的浅灰占位。

### 7.2 筛选能力

文章列表支持组合筛选：

- 分类：由导航栏“文章”子菜单带入。
- 关键词：匹配文章标题、摘要或后端约定的公开检索字段。
- 时间：按年份与月份筛选，体现博客归档语义。
- 标签：从公开标签集合中选择，允许同时选择多个标签，多个标签之间使用 OR 语义。
- 页码：按后端分页结果切换。

筛选界面应面向访客，不得照搬后台管理平台：

- 不使用密集表单栅格。
- 不堆叠“查询”“重置”等后台按钮矩阵。
- 不展示分类横栏、分类标签组或分类下拉框；分类只能从导航栏“文章”子菜单选择。
- 关键词使用轻量搜索框。
- 年月使用阅读友好的归档选择。
- 筛选区不单独占用一整行展示标签气泡，只保留“更多标签”触发器。桌面端点击后在触发器附近打开轻量弹窗，所有可选标签气泡都放在弹窗内；移动端使用承载同一内容的底部抽屉。
- 弹窗内每个气泡代表一个真实标签，点击切换选中状态，允许同时点亮多个标签；气泡保持一致的高度和文字层级，不得制作成标签云或按热度改变大小。
- 弹窗提供明确的关闭方式，点击外部或按 `Escape` 关闭。选择标签后可以立即更新结果，弹窗保持打开，便于连续多选；关闭弹窗不清除已选条件。
- 多个已选标签使用 OR 筛选：文章只要包含任意一个已选标签即可进入结果集。
- 当前条件以可移除的简洁摘要展示，并提供清空全部条件的入口。已选标签只在该摘要中回显，每个标签可以单独移除，不在常驻筛选区重复渲染气泡；移除分类条件或清空全部条件后进入不带 `categoryId` 的 `/articles`。

#### 7.2.1 标签数量增长

- 标签弹窗不能随标签数量无限增高。桌面端宽度建议约 `360px`，最大高度不超过 `min(420px, 60vh)`；移动端底部抽屉最大高度不超过 `70vh`。
- 标签总数不超过 `12` 个时直接展示全部气泡，不额外显示搜索框。
- 标签总数超过 `12` 个时，在弹窗顶部显示轻量的“搜索标签”输入框。搜索只在已经加载的公开标签集合中按名称即时过滤，不发送新的接口请求，也不搜索文章正文。
- 已选标签集中显示在弹窗顶部的吸附区域，并提供“清空已选”文本操作；同一标签不得同时出现在已选区和未选区。关闭弹窗后，已选状态仍由 URL 中的 `tagIds` 恢复。
- 未选标签气泡放在独立的可滚动区域中，保持后端返回顺序，不按热度、文章数量或前端推断重新排序。滚动弹窗不得带动页面主体滚动。
- “更多标签”触发器在存在已选标签时显示选中数量，例如“更多标签 · 2”，但不在常驻筛选区重复展开标签名称。
- 标签名称过滤无结果时，在弹窗内显示“没有匹配的标签”，保留搜索输入和已选区，不关闭弹窗、不清除条件。
- 弹窗使用多选语义并支持键盘操作：触发器可通过 `Enter` 或空格打开，气泡可通过 `Tab` 聚焦并使用 `Enter` 或空格切换，`Escape` 关闭后焦点返回“更多标签”触发器。

筛选状态写入 URL：

```text
/articles?categoryId=2&tagIds=5,8&year=2026&month=7&keyword=樱花&page=2
```

`tagIds` 使用逗号分隔的标签 ID，写入 URL 前必须去重并按数字升序排列。刷新、浏览器前进后退和直接访问必须能恢复相同筛选状态。增减任一标签或改变其他筛选条件后，页码重置为第一页。

### 7.3 结果列表

- 使用宽松的左右交错图文行，强调封面、标题和摘要，不使用整齐重复的同向列表。
- 第一篇文字在左、封面在右；第二篇封面在左、文字在右；后续按奇偶顺序继续交替。
- 无论封面位于哪一侧，标题、摘要和元数据都保持左对齐，不随布局改成右对齐。
- 每行保持一致的封面比例、信息层级和基本高度，使用纵向留白与短轨道分隔符建立节奏，不制作成瀑布流或独立浮动卡片。
- 展示分类、发布时间、浏览数、评论数和点赞数。
- 列表不要求逐篇返回或展示标签集合，标签只作为筛选条件。
- `isTop=true` 显示轻量“置顶”标记。
- `isRecommend=true` 显示“精选”标记。
- 排序由后端决定，前端不重新排序。
- 底部使用简洁页码分页，切页后滚动到结果区顶部。
- 列表行首次进入视口时，从封面所在方向轻移约 `20px` 并淡入；分页切换后只为新结果执行一次，不能在滚动返回时反复跳动。

### 7.4 无结果与失败

- 无结果时保留筛选条件，提示访客修改条件或清空筛选。
- 请求失败时保留筛选状态并提供重试。
- 分类或标签接口单独失败时，不阻断已经可以展示的文章结果。

## 8. 文章详情阅读页设计

### 8.1 阅读结构

- 沿用统一导航和轻量横幅，详情页媒体层直接使用文章封面；没有封面时使用浅灰色占位。详情页仍复用同一动态双层波浪组件。
- 文章封面只出现在详情页顶部横幅，不重复渲染到正文阅读区；正文中的图片仅来自作者 Markdown 内容。
- 桌面端采用左侧正文、右侧阅读工具的双栏结构。正文阅读宽度约 `720px`，右侧阅读工具宽度约 `280px`；重点图片可以适度突破正文宽度，但不能进入右侧工具列。
- 右侧工具列滚动到指定位置后吸附在视口顶部。工具列自身不得覆盖正文；目录过长时只允许目录区域内部滚动。
- 标题下展示分类、标签、发布时间、浏览数、评论数和点赞数。
- 若 Markdown 正文第一项是与文章标题完全相同的一级标题，展示层只保留页面头部标题，正文不重复渲染该一级标题。
- 精选文章显示“精选”，不显示“推荐”。
- 正文支持标题、段落、引用、图片、列表、表格、代码块和链接。
- 正文只应用基础阅读排版，不添加樱花主题文字特效；后台编辑器写入并经过现有清理边界允许的颜色、加粗、高亮、标题和引用格式必须优先展示。
- 提供阅读进度提示和返回文章列表入口。

### 8.2 阅读工具

- 右侧工具列固定包含文章目录、阅读百分比、字号、行高、正文宽度和返回顶部。
- 文章目录根据正文 `H2` 与 `H3` 自动生成，点击目录项滚动到对应标题，当前章节沿细轨道高亮移动。
- 字号、行高和正文宽度属于当前设备的阅读偏好，可以通过统一的本地存储工具保存，不依赖登录或后端。
- 阅读设置必须使用明确的分段控件、步进控件或图标按钮，不堆叠文字胶囊按钮。
- 阅读进度顶部细线与右栏百分比使用同一进度来源，不创建两套互相不同步的计算。
- 返回顶部在离开文章开头后才可用，使用熟悉的向上箭头图标和提示文本。
- 每个代码块右上角提供复制按钮，复制成功显示短暂轻提示；代码复制属于代码块自身能力，不放入右侧工具列。
- 首期不增加页内全文搜索、沉浸模式或跨文章阅读记录。

### 8.3 点赞

- 文末只提供点赞按钮，不提供分享按钮。
- 匿名访客允许点赞。
- 请求进行中禁用按钮，避免短时间重复提交。
- 成功后使用后端返回的最新点赞数量覆盖页面值。
- 触发限频时根据后端错误码和等待时间展示温和提示。
- 普通失败不擅自增加点赞数量。
- 防刷必须由后端执行；前端本地状态、按钮禁用或 `localStorage` 不能作为安全措施。

点赞首期为一次性动作，不提供取消点赞。

### 8.4 评论边界

本期只显示后端返回的 `commentCount` 数字。不得出现：

- 评论区标题。
- 评论输入框或提交按钮。
- 评论列表、头像或内容。
- 回复、排序、加载更多。
- 评论审核或管理入口。
- 带边框的“评论功能开发中”占位模块。

正文结尾和点赞操作后保持自然留白，不暗示已经实现评论模块。

### 8.5 详情异常状态

- 文章不存在、未发布或已隐藏时不渲染残缺正文。
- 网络失败提供重试。
- 正文内容为空时显示明确的内容不可用状态。

## 9. 图片与占位规范

### 9.1 数据驱动规则

图片组件优先使用接口返回的图片地址。仅在以下情况显示占位：

- 图片 URL 为空或缺失。
- 图片请求真实加载失败。
- 正式站点级视觉资产尚未配置。

一旦接口返回新的有效图片 URL，组件必须重置旧错误状态并重新加载，成功后自动替换占位，不需要修改前端代码。

品牌 Logo、主题图标、鼠标花瓣和状态插图属于版本控制内的站点级静态资产，不从文章接口读取；它们必须使用明确文件清单和集中资产入口，禁止散落硬编码路径。

### 9.2 占位样式

- 使用统一中性浅灰纯色，不使用随机图片、外链图库、渐变、图标或二次元假素材。
- 中性浅灰只表示正式媒体资源缺失时的技术占位，不是前台主题色、配色参考或最终头图效果；文字和图标设计必须以正式樱花车站视觉方向为准。
- 保持头图、文章封面和正文图片的固定比例与稳定尺寸，避免布局跳动。
- 默认占位颜色集中定义为 CSS 变量，不在多个组件重复写色值。
- 不在占位块上显示“图片加载中”等可见文字。
- 必要的加载失败说明通过可访问性文本提供。

原型图中的文章图片只表达最终视觉方向，不得直接裁切为正式内容资产。

## 10. 避免硬编码规范

- 首页精选、最新文章、文章封面和统计数量来自公开接口。
- 分类名称、数量和顺序来自公开分类接口，除“全部文章”外不写死。
- 标签名称和文章数量来自公开标签接口，不预置热门标签。
- 可用年份和月份来自文章元数据接口，不在前端写死年份范围。
- 精选只由 `isRecommend` 决定，置顶只由 `isTop` 决定。
- 文章排序由后端决定，前端不二次排序。
- 图片显示由 URL 和加载状态驱动，不能给特定文章固定绑定占位。
- 筛选状态以 URL 查询参数为可恢复来源。
- API 路径和参数转换集中在 `src/api/`，页面禁止拼接接口地址。
- 分页大小使用集中常量。
- 颜色、间距、圆角、断点、占位色、花瓣密度和动画时长使用统一变量。
- 正式站点名称固定为“梦见樱町”。站点名称、欢迎语和头图最终应来自公开站点配置；接口完成前集中放在单一前台配置模块，不得散落在 JSX 和 CSS。导航、首页标题、浏览器标题和无障碍名称必须读取同一配置值。
- 固定界面文案可以保留，例如“首页”“文章”“暂无文章”；这不属于业务数据硬编码。

## 11. 公开接口需求

所有公开接口沿用现有响应包装：

```ts
type ApiResult<T> = {
  code: string
  msg: string
  data: T
}
```

前台不得调用 `/admin/...` 接口。公开文章接口只返回 `published` 且允许公开访问的数据。

实现时以已核对的后端公开契约为准：`PublicArticleListVO` 和 `PublicArticleDetailVO` 的 `isTop`、`isRecommend` 为可空布尔值，统计字段为可空整数，`publishedAt` 为 ISO-8601 带时区时间字符串；前端展示层须对缺失值做中性降级，不得将空值误判为精选或置顶。分页查询的 `tagIds` 通过逗号分隔字符串传递给后端 `List<Long>` 参数，空数组不发送该参数，年月筛选只有在 `year` 有效时才允许发送 `month`。匿名点赞成功数据为 `{ likeCount: number }`；限频错误码为 `429`，结构化错误数据为 `{ retryAfterSeconds: number }`，请求层须向后兼容地保留该详情。

### 11.1 首页

```http
GET /api/home
```

建议响应：

```ts
type HomeData = {
  featuredArticles: PublicArticleListItem[]
  latestArticles: PublicArticleListItem[]
}
```

后端负责精选和最新内容的数量、顺序与公开状态过滤。前端不补假数据。

### 11.2 文章分页

```http
GET /api/articles
```

查询参数：

```ts
type PublicArticlePageParams = {
  page: number
  size: number
  keyword?: string
  categoryId?: number
  tagIds?: number[]
  year?: number
  month?: number
}
```

响应使用现有 `PageResult<PublicArticleListItem>`：

```ts
type PublicArticleListItem = {
  id: number
  title: string
  summary?: string | null
  coverUrl?: string | null
  categoryId?: number | null
  categoryName?: string | null
  isTop: boolean
  isRecommend: boolean
  viewCount: number
  commentCount: number
  likeCount: number
  publishedAt: string
}
```

后端负责组合筛选、排序和分页。`tagIds` 为空时不按标签过滤；存在多个值时使用 OR 语义，只要文章关联任意一个标签即可匹配。列表不要求返回逐篇标签集合。

### 11.3 文章详情

```http
GET /api/articles/{id}
```

响应至少包含：

```ts
type PublicArticleDetail = PublicArticleListItem & {
  content: string
  tags: Array<{
    id: number
    name: string
  }>
}
```

后端按既定规则统计浏览量。草稿、隐藏或不存在的文章不能返回公开正文。

### 11.4 公开分类

```http
GET /api/categories
```

只返回可见分类，并按 `sortOrder` 排序。响应至少包含 `id`、`name`、`articleCount`。

### 11.5 公开标签

```http
GET /api/tags
```

响应至少包含 `id`、`name`、`articleCount`，用于文章页筛选。

### 11.6 文章筛选元数据

```http
GET /api/articles/meta
```

建议响应：

```ts
type ArticleFilterMeta = {
  archives: Array<{
    year: number
    months: Array<{
      month: number
      articleCount: number
    }>
  }>
}
```

该接口用于生成有内容的年份和月份选项，避免前端写死年份或展示空月份。

### 11.7 点赞

```http
POST /api/articles/{id}/like
```

请求体为空。成功响应：

```ts
type ArticleLikeResult = {
  likeCount: number
}
```

后端必须负责：

- 点赞幂等或同一访客重复点赞识别。
- 按文章、访客标识、IP 和时间窗口执行限频策略。
- 返回稳定的重复点赞与限频错误码。
- 限频时提供 `retryAfterSeconds`，便于前端展示等待时间。
- 不信任前端按钮状态或本地缓存作为防刷依据。

### 11.8 可选站点配置

为了最终消除站点级内容硬编码，后端后续可以提供：

```http
GET /api/site-config
```

可包含站点名称、欢迎语和首页头图 URL。该接口不是首期页面联调的强制前置条件，但在正式站点配置进入后台管理后应接入。Live2D 不属于本期站点配置；以后新增时必须先补充独立设计。

## 12. 数据流与缓存

- 页面通过 `src/queries/` 的 TanStack Query hooks 获取数据。
- 首页、文章分页、文章详情、分类、标签和筛选元数据使用独立 Query Key。
- 文章分页 Query Key 必须包含规范化后的全部筛选参数；`tagIds` 必须去重并按数字升序排列后再进入 Query Key。
- URL 参数变化驱动文章分页查询。
- 点赞成功后更新当前详情缓存中的 `likeCount`，并使相关首页与列表查询失效或同步更新。
- 点赞失败不修改缓存计数。
- 分类或标签失败不清空其他已成功数据。

## 13. 响应式设计

- 宽桌面页面头部全宽；首页保持中央内容列，文章列表使用可容纳交错图文的加宽内容区，文章详情使用正文与工具列组成的双栏阅读区。
- 桌面视口缩小时优先收窄页面外栏，不能压缩正文到不可读。
- 桌面端首页保持非对称精选布局。
- 平板端收窄为两栏并减少装饰，不压缩正文可读性。
- 移动端按主精选、次精选、最新文章顺序改为单列叙事。
- 文章列表筛选在小屏按关键词、年月、标签纵向排列。
- 文章列表在移动端取消奇偶交错，统一使用封面在上、标题摘要和元数据在下的竖屏单列结构。
- 文章详情在小屏收起右侧吸附工具列，通过底部抽屉打开目录和阅读设置；正文保持安全边距、代码横向滚动和图片不溢出。
- 全屏花瓣在移动端降低粒子数量和回旋幅度，不能因屏幕变窄而提高正文区域的视觉密度。
- 所有固定格式区域使用稳定比例或尺寸约束，动态内容不得造成布局跳动。

## 14. 加载、空数据与错误状态

- 首页精选与最新文章分别处理加载和失败，单区失败不阻断另一块内容。
- 首页没有任何文章时显示整体空状态。
- 文章列表加载时保持筛选区稳定。
- 文章列表无结果时保留条件并提供修改或清空入口。
- 分类、标签和归档元数据失败时禁用对应筛选并允许重试。
- 详情加载失败允许重试。
- 路由切换显示樱花轨道 Logo 圆环过渡，真实页面就绪后立即收尾；不得用固定等待时间伪造加载。
- 空数据与请求失败使用已确认的空车票和暂停车信号状态图，图像失败时仍保留清晰文字与重试操作，不依赖插图传达唯一语义。
- 文章不可公开访问时显示统一错误页面，不渲染旧缓存正文。
- 图片失败只影响对应图片，不使整篇文章失败。
- 点赞限频与普通网络错误使用不同提示。

## 15. 测试与验收

自动化测试至少覆盖：

- 首页公开数据加载、精选映射和精选为空时的布局降级。
- 首页与文章列表布局职责不同，不复用同一种列表结构。
- 首页站台时钟、站牌短句和单会话列车动画不依赖登录或后端数据。
- 全屏花瓣不拦截交互，关闭偏好可在当前设备恢复，页面不可见时暂停。
- 系统开启减少动态效果时不执行花瓣飘落、视差和滚动入场。
- “梦见樱町”品牌名称在导航、首页主标题、浏览器标题、favicon 名称和无障碍名称中保持一致，旧临时名称不再出现在正式界面。
- 首页樱花轨道 Logo reveal 每个浏览会话只自动播放一次；返回首页直接显示最终态，会话存储不可用、触屏或减少动态效果环境可以安全降级。
- Pixel2Motion 产出的静态 SVG 使用最低充分复杂度和稳定语义 `id`，关键帧不裁切，动画最终帧与静态验收帧一致，演示控制外壳不进入正式站点。
- 精选文章在内容已存在于 DOM 后按“轨道延伸、站点章点亮、文章错峰抵达、高光收尾”播放一次；最近文章从下往上错峰进入，快速滚动、返回页面和组件卸载不产生残留 Trigger 或不可见内容。
- 页面切换具有统一且不会阻塞导航的淡出与轻位移进入过渡。
- 页面切换圆环加载器由真实就绪状态结束，普通模式和减少动态效果模式均不会阻断导航或形成无限遮罩。
- 桌面鼠标花瓣伴随、悬停绽放和点击散落不拦截点击，不进入输入与文本选择场景，并在触屏和减少动态效果环境关闭。
- 寒蝉圆黑体在首页、列表、详情和共享导航正确加载；加载失败回退不会造成文字溢出、遮挡或布局跳动。
- 头图主标题具有暖白填充、花瓣粉描边和樱莓红偏移阴影，移动端降低效果强度且保持字形清晰。
- 正文作者格式不被主题文字样式覆盖，代码块继续使用等宽字体。
- 全前台图标在 `16px`、`24px` 和 `32px` 三档保持基线、轮廓和语义清晰，纯装饰图形不会被辅助技术重复朗读。
- 所有生成式透明资产在浅色、深色、头图和棋盘格背景上无明显白边、黑边、断线和背景残留。
- 导航分类动态加载与移动端选择后收起。
- 点击“文章”主入口进入全部文章并清除分类条件，分类子菜单不重复显示“全部文章”。
- 分类选择器只存在于导航栏“文章”子菜单，文章列表筛选区不重复渲染分类选项。
- 分类、关键词、年月、多标签和页码与 URL 双向同步。
- “更多标签”弹窗中的气泡可以连续多选，已选标签只回显到条件摘要；`tagIds` 在 URL 与 Query Key 中去重排序，并按 OR 语义请求结果。
- 标签超过 `12` 个时弹窗显示本地标签搜索、受限高度滚动区和吸附的已选区；搜索无结果、清空已选和键盘关闭行为符合规格。
- 任一筛选改变后页码回到第一页。
- 分页请求包含全部有效筛选参数。
- 置顶和精选前台文案映射。
- 桌面文章列表按奇偶行交错封面方向，移动端恢复封面在上的单列顺序。
- 图片 URL 为空时显示灰色占位。
- 图片加载失败后显示占位。
- 图片 URL 更新后重新加载并替换占位。
- 详情正文结构、标签与统计数据显示。
- 详情目录由 `H2/H3` 生成并跟随当前章节，阅读设置可保存设备偏好，代码块可以复制。
- 详情工具列桌面端吸附且不覆盖正文，移动端收进底部抽屉。
- 页面不存在、隐藏或未发布时不显示正文。
- 点赞进行中、成功、重复、限频和普通失败状态。
- 页面不存在评论模块和分享按钮。

不对颜色、间距、阴影和动画中间帧做脆弱的单元测试。完成实现后使用真实浏览器检查桌面与移动端的布局、图片裁切、文字溢出、波浪衔接、导航菜单、详情双栏、花瓣阅读衰减和页面过渡；同时检查减少动态效果设置。

正式交付前至少通过：

```text
npm run typecheck
npm run lint
npm run test:run
npm run build
```

## 17. Product Design 后续输入

Product Design 阶段必须以本文为规格，并遵守：

- 先生成首页母版，再以母版图生图生成文章列表和详情页。
- 三页头部均可全宽。首页正文居中；文章列表按奇偶行左右交错；文章详情使用左侧正文与右侧吸附阅读工具列。
- 首页必须具有“樱花轨道叙事”的个性化布局。
- 三页表现统一的全屏樱花飘落氛围，并表达页面切换动画方向；花瓣不得影响正文可读性。
- 三页使用寒蝉圆黑体和统一的日系动画 UI 图标语言；首页主标题使用暖白、花瓣粉和樱莓红构成的动画标题牌，不得从灰色占位图提取配色。
- 品牌采用樱花轨道图形标与实时文字组合，页面切换使用简化标和环形轨道加载器，桌面指针使用可降级的花瓣伴随效果。
- 首页表现站台时钟、可切换短句和单会话远景列车氛围，但不得把它们画成独立业务卡片。
- 文章列表保持阅读型筛选，不得像后台管理平台；点击“文章”主入口表示全部文章，分类子菜单只列真实分类；列表筛选区不得出现分类横栏或分类控件；常驻区只显示“更多标签”入口，原型以打开的轻量弹窗展示大小一致的多选气泡和 OR 筛选，不能额外增加标签气泡横栏；桌面端首篇封面在右、次篇封面在左并持续交替，移动端统一封面在上。
- 详情页表现右侧文章目录、阅读进度和阅读设置，没有分享和评论模块。
- 分类、标签、精选、归档、点赞和图片占位均按本文真实边界表达。
- 原型可以展示理想图片氛围；实现时无正式资产必须使用数据驱动的浅灰占位。
- 原型不得新增本文禁止提前实现的功能。

## 18. 完成标准

本设计完成后，后续工作应能分别得到：

- 三张视觉一致的首页、文章列表和文章详情原型图。
- 一套不会妨碍阅读、支持减少动态效果并可跨三页复用的车站氛围与页面过渡规范。
- 一套寒蝉圆黑字体层级、樱花轨道 Logo、全前台主题图标、鼠标花瓣和透明资产质量规范。
- 三个可公开访问、响应式且具备完整状态的页面。
- 与管理接口隔离的公开前台接口层。
- 可直接交给后端开发的公开接口需求。
- 不依赖假业务数据、不会因设置真实图片而继续显示占位的实现。
- 明确受控的首期范围，不提前制作评论、分享和其他未确认模块。
