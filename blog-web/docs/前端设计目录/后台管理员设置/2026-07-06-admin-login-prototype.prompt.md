使用 huashu-design 模板：网页风格库「斜切流体渐变带 Angled Fluid Gradient」，融合「功能主义网格社区 Functional Brutalism」的清晰表单结构。目标是年轻、干净、有轻微渐变感的后台登录页，不是企业模板。

生成一张高保真 UI 原型图，主题是「个人博客后台 - 登录页」。这是给后续 AI 前端开发参考的产品原型，不是营销海报，不是插画。

画布：横向桌面 UI mockup，1536x1024，像真实浏览器页面截图，中文文字由图像模型原生生成，不要用脚本后期叠字。

页面结构必须出现：
1. 居中登录面板，标题「后台登录」。
2. 表单字段：
   - 用户名输入框，示例值 admin。
   - 密码输入框，使用密码圆点或星号。
3. 主按钮「登录后台」，按钮可带简洁登录图标感，但不是必须带图标，不要 emoji。
4. 错误提示状态：短文案「用户名和密码不能为空」或「无法连接后端服务」。只保留一个短错误状态参考，不要堆多种错误。
5. 提交中状态参考：按钮旁或按钮内部出现「登录中」，但尺寸不跳动。
6. 小号接口标注允许出现：「POST /api/admin/login」「成功进入 /admin」，但必须像设计稿标注，不要写成长说明。

视觉方向：
- 背景为近白到浅灰，右上或左下有一条斜切流体渐变带，蓝、青、珊瑚粉，轻盈不刺眼。
- 登录面板不要做厚重企业卡片，可用细线、浅背景、少量柔和阴影。
- 表单布局极简但完整，标签清楚，错误文案位置固定。
- 页面要年轻、有空气感，同时保持后台登录的可信赖感。

严格边界：
- 不出现注册入口。
- 不出现找回密码。
- 不出现第三方登录。
- 不出现营销文案、产品介绍、功能卖点。
- 不出现后台首页、设置页内容。
- 不要大段说明，不要把页面做成教程。

开发参考重点：
- 输入框、按钮、错误提示、加载态、登录成功跳转这些状态要能被后续开发者一眼看懂。
- 文案短、状态明确、组件边界清楚。
