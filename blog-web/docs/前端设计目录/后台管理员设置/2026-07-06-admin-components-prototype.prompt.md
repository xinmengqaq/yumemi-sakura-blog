使用 huashu-design 模板：网页风格库「斜切流体渐变带 Angled Fluid Gradient」，融合「功能主义网格社区 Functional Brutalism」的组件网格。目标是年轻、清晰、可复用的后台组件大全原型图，不是企业设计系统海报。

生成一张高保真 UI 原型图，主题是「个人博客后台 - 可复用组件大全」。这是给后续 AI 前端开发参考的组件清单原型，不是营销海报，不是插画。

画布：横向桌面 UI mockup，1536x1024，像真实设计系统页面截图，中文文字由图像模型原生生成，不要用脚本后期叠字。

最重要的边界：这是一张组件大全，不是后台功能菜单图。禁止出现「系统设置」「站点设置」「菜单管理」「角色管理」「文章管理」「分类管理」「评论管理」「版本」「更新时间」「邮箱」「创建时间」等文档未定义内容。AdminShell 缩略结构里侧边栏只允许出现「后台首页」「管理员设置」两个菜单。

必须展示这些组件，并按清晰网格分区：
1. Button：
   - primary「保存资料」
   - secondary「校验 Token」
   - danger「确认退出」
   - ghost「取消」
   - link「查看」
   - loading「登录中」
   - disabled「不可用」
   - 尺寸 sm / md 简短对比。
2. Input 与 FormField：
   - 默认输入框
   - 只读输入框「admin」
   - 密码输入框
   - 错误态「管理员名称不能为空」
   - label、输入控件、错误文案位置统一。
3. Alert：
   - success「资料已保存」
   - error「无法连接后端服务」
   - warning「Token 已过期」
   - info「Token 未校验」
4. Modal / ConfirmDialog：
   - 小型弹窗，标题「确认退出」
   - 正文「退出后需要重新登录」
   - 按钮「取消」「确认退出」
5. LoadingState / ErrorState / EmptyState：
   - 加载中
   - 加载失败「重试」
   - 暂无数据
6. PageHeader / DataSection：
   - PageHeader 标题「管理员设置」，右侧操作按钮。
   - DataSection 示例标题「管理员资料」，内容区域和底部操作区。
7. AdminShell 组件缩略结构：
   - AdminTopbar、AdminSidebar、AdminNavItem、内容区的线框示意。
   - AdminSidebar 只画两个菜单：「后台首页」「管理员设置」，其中「管理员设置」高亮。
   - 内容区只用线框占位，不放业务数据，不放系统设置类菜单。

视觉方向：
- 整体是组件板，不是长文说明页。使用浅灰底、细线卡片、少量斜切蓝青渐变作为顶部标题带或分区强调。
- 每个组件都要看起来可直接转成 React 组件。
- 图标感统一为 lucide 风格的细线图标，只用于按钮、菜单、状态提示等符号化场景；头像、上传预览、真实内容预览必须使用图片或图片占位，不要手绘复杂 SVG，不要 emoji。
- 小圆角，浅阴影或无阴影，优先细线分隔。
- 布局紧凑但不拥挤，组件名称和状态标签清楚。

文字说明要求：
- 允许短标签：「loading」「disabled」「readOnly」「error」「success」。
- 允许短开发标注：「组件不调接口」「页面负责状态」。
- 不要大段说明，不要写教程。

严格避免：
- 不要生成文章管理、统计图、评论管理等后续模块。
- 不要生成系统设置、站点设置、菜单管理、角色管理、版本号、更新时间。
- 不要把组件大全做成品牌海报。
- 不要过度企业化，不要厚重蓝灰后台模板。
- 不要过多解释性文字。
