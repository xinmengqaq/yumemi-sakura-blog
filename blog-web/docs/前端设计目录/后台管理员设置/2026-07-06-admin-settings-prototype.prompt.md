使用 huashu-design 模板：网页风格库「斜切流体渐变带 Angled Fluid Gradient」，融合「功能主义网格社区 Functional Brutalism」的信息结构，但整体更年轻、更轻盈、有渐变感，不做传统企业后台。

生成一张高保真 UI 原型图，主题是「个人博客后台 - 管理员设置整页」。这是给后续 AI 前端开发参考的产品原型，不是营销海报，不是官网，不是插画。

画布：横向桌面 UI mockup，1536x1024，像真实浏览器页面截图，清晰可读，中文文字由图像模型原生生成，不要用脚本后期叠字。

视觉方向：
- 年轻但克制：近白浅灰底，顶部和关键操作区域使用斜切流体渐变带，颜色为清透蓝、薄荷青、轻微珊瑚粉，不要大面积紫色霓虹。
- 结构必须整齐：细灰线分割、清晰表单网格、统一按钮尺寸、小圆角，避免过度企业后台模板。
- 信息密度适中：美观、整齐、便捷、简洁共存，不让任何一项压倒其他。
- 不要装饰性插画，不要假统计，不要文章、分类、标签、评论模块。

页面结构必须出现：
1. 顶部栏：左侧「个人博客后台」，右侧「当前管理员：梦梦（admin）」和「退出登录」按钮。
2. 左侧边栏：两个菜单「后台首页」「管理员设置」，其中「管理员设置」为当前高亮。active 状态只改变颜色、背景或边线，不改变尺寸。
3. 内容区标题：「管理员设置」，右侧可以有轻量状态标签「Token 未校验」。
4. 第一区块「管理员资料」：
   - 只读字段：用户名 admin，角色 admin。
   - 可编辑字段：管理员名称 梦梦，头像地址 /files/avatar.png。
   - 主按钮「保存资料」。
   - 小提示或状态条：资料已同步 / 保存失败占位，但文案要短。
5. 第二区块「修改密码」：
   - 输入项：旧密码，新密码。
   - 按钮「修改密码」。
   - 明确表现密码修改成功后需要重新登录，但不要写长段解释。
6. 第三区块「登录状态」：
   - Token 状态：未校验 / 有效 / 已刷新 三种状态以 pill 或小状态块展示。
   - 两个按钮：「校验 Token」「刷新 Token」。
7. 页面底部或右侧允许有紧凑的「确认退出」弹窗小样，标题「确认退出」，正文一句「退出后需要重新登录」，按钮「取消」「确认退出」。弹窗要像组件状态参考，不要抢主页面。

可见原型说明：
- 图中允许出现少量小号说明标注，例如「只读」「必填」「成功后回登录页」「401 清理登录态」。
- 说明文字必须短，不要把用户当新手，不要写解释性长段。

风格细节：
- 按钮可按需要包含简洁线性图标感，但不是所有按钮都必须带图标，不要 emoji。
- 头像预览必须表现为图片或纯灰色默认头像，不用图标替代。
- 输入框 1px 灰线，聚焦态可以用蓝青渐变细边。
- 危险操作只使用红色，不扩散到其他区域。
- 页面要像可以直接落地到 React + TypeScript + Vite 的后台 UI。

严格避免：
- 不要生成文章统计、访问量、评论数、图表仪表盘。
- 不要注册、找回密码、第三方登录。
- 不要深色发光 Bento，不要传统沉闷企业后台。
- 不要过多解释性文案，不要大段说明。
- 不要生成职责之外的页面。
