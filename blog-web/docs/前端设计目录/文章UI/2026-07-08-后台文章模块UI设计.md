# 后台文章模块 UI 设计规格

## 设计目标

本规格用于设计个人博客后台第四阶段的文章模块 UI。

第四阶段只接入文章主表基础增删改查能力，目标是让管理员能够在后台完成文章列表查看、新建、编辑、删除和正文编辑，并真实验证后端文章接口。

文章正文编辑器采用类似飞书文档的块状编辑体验，但第一版只作为当前项目内嵌组件使用。编辑器内部按未来可抽离开源的边界设计，避免和博客后台业务、登录态、路由、接口请求直接耦合。

本阶段不做普通 textarea 式 Markdown 编辑器，也不做源码视图。用户在正文区看到的是所见即所得的块状内容，底层保存给后端的仍然是 Markdown 字符串。

## 后端依据

本规格以 `D:/daimai/项目学习/springboot/` 中真实后端第四阶段文章模块为准。

已确认的文章接口如下：

| 能力         | 方法   | 地址                         | 说明                         |
| ------------ | ------ | ---------------------------- | ---------------------------- |
| 分页查询文章 | GET    | `/api/admin/articles`        | 支持页码、每页数量、关键词、状态 |
| 查询文章详情 | GET    | `/api/admin/articles/{id}`   | 根据文章 ID 查询详情         |
| 新增文章     | POST   | `/api/admin/articles`        | 新增草稿、已发布或隐藏文章   |
| 修改文章     | PUT    | `/api/admin/articles/{id}`   | 修改文章基础字段             |
| 删除文章     | DELETE | `/api/admin/articles/{id}`   | 根据文章 ID 删除文章         |

统一响应结构继续沿用项目已有规则：

```json
{
  "code": "200",
  "msg": "请求成功",
  "data": {}
}
```

文章保存参数按第四阶段 UI 收口为：

```ts
type ArticleSaveParams = {
  title: string
  summary?: string
  content: string
  coverUrl?: string
  status: 'draft' | 'published' | 'hidden'
}
```

后端 DTO 中已经存在但第四阶段 UI 不展示、不编辑的字段：

```text
categoryId
tagIds
isTop
isRecommend
```

这些字段分别属于后续分类标签、置顶推荐和状态流转阶段。本阶段不提前产品化，避免前端界面超出后端第四阶段学习闭环。

## 页面范围

本阶段设计以下页面：

| 页面         | 路由                       | 说明                 |
| ------------ | -------------------------- | -------------------- |
| 文章列表页   | `/admin/articles`          | 横向卡片列表和筛选   |
| 新建文章页   | `/admin/articles/new`      | 新建文章和正文编辑   |
| 编辑文章页   | `/admin/articles/:id/edit` | 编辑已有文章和删除入口 |

后台侧边栏新增一个菜单：

| 菜单     | 路由              | 说明       |
| -------- | ----------------- | ---------- |
| 文章管理 | `/admin/articles` | 管理文章主表 |

## 非目标

本阶段不做：

- 分类选择。
- 标签选择。
- 置顶设置。
- 推荐设置。
- 批量删除。
- 文件上传。
- 正文图片上传。
- Markdown 源码视图。
- 编辑器源码抽屉。
- 编辑器属性抽屉。
- 自动保存。
- 移动端适配。
- 前台文章展示页。

## 视觉主题

文章模块继续沿用已有后台主题：功能主义灰线后台 v2。

视觉目标：

- 清楚。
- 稳定。
- 可扫读。
- 适合内容管理。
- 不做传统沉闷表格后台。
- 不用假统计和假内容填充页面。
- 编辑器要有写作空间，但仍然属于后台工作台。

文章列表不使用表格，使用横向列表卡片。原因是文章属于内容对象，封面、标题、摘要和状态更适合卡片表达；同时横向卡片仍保留后台扫描效率，不会变成前台博客卡片墙。

后台文章页面只按桌面端管理工作台设计和验收，不做手机端或小屏专用布局。

设计稿还原时以真实桌面端浏览器工作区尺寸为准，优先还原正常后台页面在浏览器中的宽度和留白关系，不按参考截图文件本身的像素尺寸机械复刻，避免把页面压成偏窄比例。

## 文章列表页

文章列表页用于查询、筛选、进入编辑和删除文章。

布局如下：

```text
+------------------------------------------------------+
| 文章管理                         [新建文章]          |
| 管理文章主表基础内容，不包含分类和标签。              |
+------------------------------------------------------+
| 关键词 [________________] 状态 [全部 v] [查询] [重置] |
+------------------------------------------------------+

+------------------------------------------------------+
| [封面图]  Spring Boot 博客后台学习记录      [已发布] |
|          记录文章模块从数据库表到接口闭环的学习过程。 |
|          阅读 128 · 评论 3 · 发布 2026-07-07          |
|          更新 2026-07-07                 [编辑] [删除] |
+------------------------------------------------------+

+------------------------------------------------------+
| [占位]    我的第一篇草稿文章                 [草稿]   |
|          这是一篇还没有发布的 Markdown 草稿。          |
|          阅读 0 · 评论 0 · 尚未发布                   |
|          更新 2026-07-07                 [编辑] [删除] |
+------------------------------------------------------+

+------------------------------------------------------+
| 共 12 条                                      < 1 / 2 > |
+------------------------------------------------------+
```

筛选区：

| 控件   | 规则                                      |
| ------ | ----------------------------------------- |
| 关键词 | 对应 `keyword`，最多 50 字符              |
| 状态   | 全部、草稿、已发布、隐藏，对应 `status`   |
| 查询   | 使用当前筛选条件请求列表                  |
| 重置   | 清空关键词和状态，回到第一页              |

横向文章卡片展示：

| 内容     | 规则                                                   |
| -------- | ------------------------------------------------------ |
| 封面     | 有 `coverUrl` 显示图片；为空或加载失败显示封面占位     |
| 标题     | 显示真实标题，不使用假标题                             |
| 摘要     | 有摘要显示摘要；无摘要显示“暂无摘要”                   |
| 状态     | 显示草稿、已发布、隐藏三种状态标签                     |
| 阅读数   | 显示 `viewCount`                                       |
| 评论数   | 显示 `commentCount`                                    |
| 发布时间 | 有 `publishedAt` 显示发布时间；为空显示“尚未发布”      |
| 更新时间 | 显示 `updatedAt`                                       |
| 操作     | 编辑、删除                                             |

空状态和错误状态：

- 列表为空时显示 `EmptyState`。
- 筛选无结果时显示筛选无结果文案，并提供重置入口。
- 加载失败时显示 `ErrorState` 和重试入口。
- 删除失败时显示后端 `msg`。

删除交互：

```text
点击文章卡片删除
→ 打开确认弹窗
→ 弹窗显示文章标题
→ 确认后调用 DELETE /api/admin/articles/{id}
→ 成功后刷新列表或移除当前卡片
→ 失败显示后端 msg
```

## 新建与编辑页

新建页和编辑页共用同一套页面结构。

编辑页采用左侧正文编辑器、右侧固定文章信息栏。比例为左侧约 70%，右侧约 30%。顶部只放返回入口和保存状态，不放大堆操作按钮。

这里的“块状正文编辑器”指一篇标准文档内部的块级内容结构，不是把每个块默认做成独立卡片或独立表单。标准态视觉应优先呈现一块连续的文档编辑画布，内容从上到下自然排版，块状态只在当前文档中的局部位置出现。

布局如下：

```text
+------------------------------------------------------+
| ← 返回文章列表                         保存状态：未保存 |
+------------------------------------------------------+
| 正文块编辑器区域                         | 文章信息       |
| -------------------------------------- | -------------- |
| + 输入 / 或 Markdown 快捷语法创建块     | 标题 [______]  |
|                                        | 摘要 [______]  |
| 一级标题块                              | 封面 URL       |
| 段落块文字内容                          | [封面预览]     |
| 引用块                                  | 状态           |
| 代码块                                  | (o) 草稿       |
|                                        | ( ) 已发布     |
| [块工具浮层] [文字工具浮层]             | ( ) 隐藏       |
|                                        | [保存文章]     |
|                                        |                |
|                                        | 危险区         |
|                                        | [删除文章]     |
+------------------------------------------------------+
```

右侧文章信息栏字段：

| 字段   | 规则                                |
| ------ | ----------------------------------- |
| 标题   | 必填，最多 120 字符                 |
| 摘要   | 可空，最多 300 字符                 |
| 封面   | 只支持 URL，最多 500 字符           |
| 状态   | 草稿、已发布、隐藏，默认草稿        |
| 保存   | 点击后按当前状态保存文章            |
| 删除   | 编辑已有文章时显示，新建页不显示    |

保存交互：

```text
填写标题、摘要、封面、状态和正文
→ 点击「保存文章」
→ 前端校验标题和正文
→ 新建页调用 POST /api/admin/articles
→ 编辑页调用 PUT /api/admin/articles/{id}
→ 成功后显示「已保存」
→ 新建成功后使用后端返回的 id 进入 `/admin/articles/:id/edit`
→ 失败显示后端 msg
```

保存状态：

| 状态       | 说明                         |
| ---------- | ---------------------------- |
| 未修改     | 当前内容和最近一次保存结果一致 |
| 有未保存修改 | 用户改动后尚未保存           |
| 保存中     | 正在提交，禁止重复点击       |
| 已保存     | 最近一次保存成功             |
| 保存失败   | 最近一次保存失败             |

本阶段不做自动保存。只有点击「保存文章」才调用后端接口。

编辑页删除交互：

```text
点击右侧危险区「删除文章」
→ 打开确认弹窗
→ 弹窗显示文章标题
→ 确认后调用 DELETE /api/admin/articles/{id}
→ 成功后返回文章列表
→ 失败显示后端 msg
```

## 块状 Markdown 编辑器定位

编辑器名称在当前仓库内暂定为 `BlockMarkdownEditor`。

定位：

- 当前阶段服务文章正文编辑。
- 未来可以抽离为独立开源项目。
- 编辑器对外接收 Markdown 字符串。
- 编辑器对外输出 Markdown 字符串。
- 编辑器内部维护块模型。
- 编辑器 UI 是所见即所得块编辑，不显示 Markdown 源码。

对外使用方式：

```tsx
<BlockMarkdownEditor
  value={content}
  onChange={setContent}
/>
```

编辑器不能直接依赖：

- 后台路由。
- 文章接口。
- TanStack Query。
- Zustand 登录态。
- 管理员布局。
- 文章状态。
- 后台页面样式类名。

编辑器可以使用项目 CSS 变量，但必须保持可抽离。

## 编辑器底层数据规则

后端仍只保存 `content: string`。

保存格式：

```text
普通段落、标题、引用、列表、任务列表、代码块、图片、分割线：保存为 Markdown 语法
文字颜色和背景高亮：保存为受限 HTML span
复杂表格：保存为受限 HTML table
```

这意味着底层仍然是 Markdown 文档，只是允许 Markdown 中混写编辑器白名单内的 HTML。

内部模型：

```text
顶层：扁平块列表
块内：按块类型维护子结构
```

视觉表现规则：

- 块属于同一篇文档的连续内容流，共享一个主编辑画布。
- 标题、段落、引用、列表等普通块默认不加独立外框，不做一块一个输入卡片。
- 代码块、表格块可以保留自身语义容器，但仍然属于连续文档中的局部结构，不代表整个编辑区采用卡片堆叠。
- 块的悬停、选中、插入、拖拽等状态是文档内局部状态，跟随当前块出现，不做整页默认全量显式展示。

选择扁平块列表的原因：

- 更适合飞书式块状编辑体验。
- 更容易做块工具浮层、块移动、块复制和块删除。
- 比完整递归 AST 更容易在第四阶段落地。
- 未来抽离开源时，核心模型边界更清楚。

复杂块规则：

- 列表块内部支持最多两级缩进。
- 表格块内部维护行、列、单元格、合并、列宽和对齐。
- 代码块内部维护语言和代码文本。
- 图片块内部维护 URL 和替代文本。

## Markdown 能力范围

第一版支持 CommonMark + GFM 常用集：

- 标题。
- 段落。
- 引用。
- 无序列表。
- 有序列表。
- 任务列表。
- 代码块。
- 行内代码。
- 加粗。
- 斜体。
- 下划线。
- 删除线。
- 链接。
- 图片。
- 分割线。
- 表格。
- 文字颜色。
- 背景高亮。

不支持：

- 数学公式。
- Mermaid 图。
- 脚注。
- 目录锚点。
- 任意 HTML。
- iframe、video、audio 等嵌入内容。

## 块创建方式

编辑器支持快捷输入和可发现的加号菜单。

创建方式：

```text
输入 / 打开块类型菜单
点击空行或块旁 + 号打开新增菜单
输入 Markdown 快捷语法自动转换块
```

Markdown 快捷转换：

| 输入         | 转换结果     |
| ------------ | ------------ |
| `# `         | 一级标题     |
| `## `        | 二级标题     |
| `### `       | 三级标题     |
| `- `         | 无序列表     |
| `1. `        | 有序列表     |
| `- [ ] `     | 任务列表     |
| `> `         | 引用         |
| 三个反引号   | 代码块       |
| `---`        | 分割线       |

## 块工具浮层

选中块时显示块工具浮层。

块工具能力：

```text
切换段落
切换标题 H1 / H2 / H3 / H4
切换引用
切换无序列表
切换有序列表
切换任务列表
插入代码块
插入图片 URL
插入表格
插入分割线
上移块
下移块
复制块
删除块
```

块工具浮层规则：

- 不改变块尺寸导致页面跳动。
- 鼠标离开和按 Esc 可以关闭。
- 当前块不可执行的操作置灰或隐藏。
- 删除块属于危险动作，但不需要弹全局确认；删除整篇文章才需要确认弹窗。

## 文字工具浮层

选中文字时显示文字工具浮层。

文字工具能力：

```text
加粗
斜体
下划线
删除线
行内代码
设置链接
取消链接
文字颜色
背景高亮色
清除格式
```

颜色规则：

- 文字颜色只能来自预设色板。
- 背景高亮只能来自预设色板。

- 保存时输出受限 HTML `span`。

示例：

```html
<span style="color:#dc2626">红色文字</span>
<span style="background-color:#fef3c7">黄色高亮</span>
```

列表支持：

```text
无序列表
有序列表
任务列表
最多两级缩进
```

列表交互：

```text
Tab 增加缩进，最多两级
```

## 表格工具

表格是第一版重点能力之一。

表格保存为 Markdown 中的受限 HTML `table`，因为合并单元格和列宽不是标准 Markdown 表格能稳定表达的能力。

表格基础能力：

```text
插入表格，默认 3x3
插入上方行
插入下方行
插入左侧列
插入右侧列
删除当前行
删除当前列
合并单元格
拆分单元格
表头开关
单元格左对齐
单元格居中
单元格右对齐
清空单元格内容
删除表格
```

表格交互：

```text
鼠标悬浮表格边缘或行列间隙时显示 + 号
点击 + 号可在当前位置插入行或列
拖拽列边界调整列宽
选中单元格或表格后，右键打开表格工具浮窗
单元格内文字复用文字工具浮层
```

表格 HTML 允许结构：

```html
<table>
  <colgroup>
    <col style="width:160px">
  </colgroup>
  <thead>
    <tr>
      <th colspan="2" style="text-align:center">标题</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td rowspan="2">内容</td>
      <td>内容</td>
    </tr>
  </tbody>
</table>
```

## 图片能力

第四阶段不做文件上传。

图片规则：

- 封面只支持 URL。
- 正文图片块只支持 URL。
- URL 为空显示图片占位。
- 图片加载失败显示错误占位和原始 URL。
- 上传能力等后端文件模块完成后再接入。

正文图片块字段：

```text
url：图片地址
alt：替代文本，可空
```

## 粘贴能力

编辑器支持三类粘贴：

| 粘贴来源 | 处理规则                           |
| -------- | ---------------------------------- |
| Markdown | 解析成块                           |
| 纯文本   | 按段落和换行拆成块                 |
| HTML     | 安全降级，只保留白名单结构和样式   |

HTML 粘贴规则：

- 白名单标签和属性保留。
- 非白名单标签移除，保留内部文本。
- `script`、`style`、`iframe`、`object`、`embed`、`video`、`audio` 直接移除。
- `onclick`、`onload` 等事件属性全部移除。
- 不认识的样式全部移除。

## 受限 HTML 白名单

允许标签：

```text
span
table
thead
tbody
tr
th
td
colgroup
col
```

允许属性：

```text
style
rowspan
colspan
```

`style` 只允许：

```text
color
background-color
text-align
width
```

限制规则：

- `color` 和 `background-color` 只能来自预设色板。
- `text-align` 只允许 `left`、`center`、`right`。
- `width` 只允许安全数值，例如 `160px` 或 `25%`。
- 不允许 `position`、`display`、`script`、`url`、`expression`。
- 不允许任意 HTML 直接进入保存结果。

## 快捷键与帮助抽屉

编辑器提供快捷键概览抽屉。

## 外部库策略

本项目自研的是块状 Markdown 编辑体验，不是从零手写 Markdown 解析器。

推荐策略：

- Markdown 解析使用成熟 Markdown AST 库。
- HTML 清理使用成熟安全库。
- 块模型、编辑 UI、浮层工具、表格交互、快捷键处理优先自研。
- 不引入完整富文本编辑器框架来替代核心编辑体验。

可考虑的依赖方向：

```text
unified / remark：Markdown 解析和序列化相关
rehype / hast：HTML AST 处理相关
DOMPurify 或同类库：HTML 清理
floating-ui 或同类小库：浮层定位
```

依赖引入原则：

- 先看现有依赖是否已经满足。
- 新依赖必须解决明确问题。
- 解析、清理、安全类依赖优先成熟稳定。
- UI 展示类依赖谨慎引入，避免破坏后台视觉一致性。
- 引入依赖时同步更新 `package.json` 和 `package-lock.json`。

## 安全与失败降级

解析失败：

- Markdown 解析失败时，不让页面白屏。
- 编辑器显示错误状态和原文恢复入口。
- 用户可以复制原始 Markdown。
- 保存按钮在解析失败时禁用，避免保存破坏性内容。

图片失败：

- 封面 URL 为空显示封面占位。
- 封面加载失败显示封面占位。
- 正文图片 URL 为空显示图片块占位。
- 正文图片加载失败显示错误占位和图片 URL。

表格失败：

- table HTML 不符合白名单时清理为普通文本或普通表格。
- 无法还原合并单元格时保留单元格文本，不保留危险属性。
- 不因为单个表格解析失败导致整篇文章不可编辑。

请求失败：

- 业务失败展示后端 `msg`。
- 网络失败展示明确的后端连接错误。
- 401 或登录过期时清理登录态并回到登录页。

## 当前后台视觉基线

文章模块必须继承现有后台重置后的冷中性工作台视觉，不再沿用旧蓝色功能主义主题或原型中的装饰风格：

- 页面使用当前后台的左对齐宽内容区、`PageHeader`、按钮、输入框、反馈状态和弹窗样式。
- 中性灰阶作为主体，低饱和赤陶红只用于当前项、链接、焦点和必要的状态强调；不使用渐变、发光和页面级阴影。
- 按钮和输入框使用 `4px` 圆角，操作容器不超过 `6px` 圆角；弹窗是唯一允许使用轻阴影的容器。
- 文章列表项采用便于扫描和重复操作的横向结构，不使用营销式大卡片、彩色徽章、图标方框或无意义统计。
- 编辑器保持连续文档画布，普通块不做成独立卡片；代码块、表格和引用只保留必要的语义视觉。
- 旧文章原型只用于确认内容结构和交互关系，不作为颜色、尺寸、圆角、阴影和页面宽度的视觉权威。

块工具浮层以当前块操作柄为定位基准，使用 `top-start`、偏移、翻转、视口避让和自动更新。浮层采用冷中性白色表面、灰色边框、`4px` 圆角和紧凑图标按钮，不使用旧蓝色主题、渐变、发光或大面积阴影；浮层作为 overlay 存在，不改变块尺寸和连续文档布局。第一个块禁止上移，最后一个块禁止下移，文档只有一个空段落时禁止删除；只读状态不显示浮层。

文字工具浮层以编辑器内非空文字选区的范围矩形作为虚拟定位基准，使用与块工具一致的紧凑视觉语言。加粗、斜体、下划线、删除线、行内代码、链接、预设文字颜色、预设背景高亮和清除格式必须作用于当前选区；颜色只能来自 `markdownSchema.ts` 的预设色板，结果必须经过现有 HTML 清理和 Markdown 序列化边界。编辑器外选区、空选区和只读状态不显示浮层。

两个浮层都只在操作时出现，`Esc` 只关闭当前浮层；关闭浮层不得清空正文、改变块顺序或破坏当前编辑位置。侧边栏“文章管理”入口已经开放，Task 13-14 不改变入口和路由行为。

列表交互直接作用于现有连续文档中的列表项：`Tab` 和 `Shift + Tab` 在 0-2 级之间调整缩进，`Enter` 在当前项后创建同级项，空列表项按 `Backspace` 退出为段落；任务列表勾选状态继续保存为 GFM 任务标记。列表项不增加卡片、工具条或额外装饰容器。

表格工具沿用冷中性浮层、灰色边框、`4px` 圆角和紧凑图标按钮。当前单元格使用低饱和赤陶红焦点边框，不使用旧参考图中的亮蓝色；工具浮层和行列插入按钮均为 overlay，不改变表格尺寸。普通点击选择当前单元格，`Shift + 点击` 扩展为矩形连续选区；至少选择两个连续单元格时允许合并，拆分作用于当前合并单元格。列宽拖拽只更新安全像素宽度，并继续通过受限 HTML 序列化和清理边界保存。

正文图片块继续只支持 URL 和可空替代文本，不增加上传入口。URL 为空显示中性占位；加载失败后显示明确错误状态和原始 URL，修改 URL 后重置失败状态并重新尝试。图片、占位和错误态使用同一 `6px` 以内圆角与灰色边框，不使用装饰阴影。

快捷键入口固定在编辑器右上角，抽屉作为右侧 overlay 显示，不压缩连续文档画布。抽屉只列出当前阶段已实现的基础编辑、块操作、文字格式、列表和表格快捷键；关闭按钮和 `Esc` 只关闭抽屉。编辑器聚焦时 `Ctrl + S` 调用外部 `onSaveShortcut`，并且不得拦截刷新、地址栏、开发者工具等浏览器或系统组合键。

文章新建和编辑页共用同一工作台布局：顶部为返回入口和保存状态，主体左侧是连续正文画布，右侧为 `360px-390px` 的 sticky 信息栏。信息栏使用无阴影、灰边框、`6px` 圆角操作面板；标题、摘要、封面 URL、封面预览、状态和保存按钮按稳定纵向顺序排列。编辑模式在面板底部显示独立危险区，新建模式不显示删除入口。

保存状态必须覆盖未修改、有未保存修改、保存中、已保存和保存失败。标题和正文为空时在对应区域显示校验反馈且不发请求；请求失败显示后端 `msg`。创建成功后替换导航到编辑路由，更新成功后保持当前页并同步最近保存快照；`Ctrl + S` 与保存按钮复用同一提交入口。删除确认必须显示当前标题，失败时保留页面内容和弹窗上下文，成功后替换导航回文章列表。

Task 19 真实浏览器验收补充：编辑文章更新成功后，详情缓存同步不得再次执行首次加载初始化并把“已保存”覆盖为“未修改”；“已保存”必须作为请求完成后的稳定状态保留，直到用户再次修改表单。

## 实施边界

- 先更新对应设计文档，再写正式功能代码。

- 接口方法放在 `src/api/article.ts`。
- 查询和变更放在 `src/queries/article.ts`。
- 类型放在 `src/types/article.ts`。
- 编辑器放在 `src/components/editor/block-markdown-editor/`。
- 页面不直接写 Axios。
- 页面不直接拼接完整后端地址。
- 页面不解析 JWT。
