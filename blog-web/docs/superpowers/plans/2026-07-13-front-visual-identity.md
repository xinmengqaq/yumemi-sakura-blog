## 1. 业务目标与完成定义

### 1.1 业务目标

将已经可用的公开博客前台从系统字体和通用图标语言升级为完整、统一、可维护的二次元樱花车站视觉识别，同时保持文章发现、筛选和阅读行为不变。

访客应能直接感知以下结果：

- 正式站名统一为“梦见樱町”，具有独立的樱花轨道 Logo，而不是临时名称或通用星形符号。
- 首页、文章列表和详情使用寒蝉圆黑与统一的樱花主题图标语言，不再依赖 Georgia、系统宋体或灰色占位图决定主题。
- 首页主标题具有清晰的动画标题牌效果，其他文字保持克制且可读。
- 页面真实等待时显示环形轨道过渡，快速缓存跳转不为展示动画被故意延迟。
- 桌面精确指针设备具有花瓣伴随、悬停绽放和点击散落效果；触屏和减少动态效果环境自动降级。
- 后台编辑器保存的受支持文字颜色、高亮、下划线和复杂表格在公开文章中安全展示，主题 CSS 不覆盖作者格式。

### 1.2 本次范围

- 寒蝉圆黑字体获取、许可留档、WOFF2 分片、自托管和前台字体层级。
- 樱花轨道 Logo、favicon、主题图标、鼠标花瓣、空状态与错误状态插图。
- `gpt-image-2` Mode A 离线生图、提示词和原始响应留档。
- 使用固定版本的 Pixel2Motion 将验收后的樱花轨道位图重建为低复杂度 SVG，并交付小体量首页品牌揭示动画及静态/动态 QA 证据。
- `rembg + birefnet-general` 离线抠图、Alpha 边缘审计和网页资产优化。
- 前台共享图标入口、品牌头部、页面状态、文章元数据和阅读工具图标统一。
- 真实路由/初次查询驱动的环形轨道过渡、既有页面进入动画和减少动态效果降级。
- 桌面鼠标花瓣效果与现有全屏花瓣动画的性能、暂停和偏好整合。
- 公开文章中后台作者格式的共享白名单、安全渲染和回归测试。
- 桌面、移动端、触屏语义、键盘、减少动态效果、字体网络请求和透明资产浏览器验收。
- 首页精选文章“樱花站台抵达”和最近文章自下而上错峰入场；内容加载与滚动揭示解耦。

### 1.3 明确排除

- 不修改数据库、后端业务、公开 API、鉴权、点赞防刷或管理后台页面。
- 不生成或接入正式首页头图、文章封面、文章插画、头像、Live2D、音频和评论功能。
- 不更换 React Router、TanStack Query、GSAP、DOMPurify 或现有 Markdown 主解析栈。
- 不把 Python、ONNX、图像模型或第三方字体服务引入浏览器运行时和生产构建。
- 不为图标系统新建通用插件平台、主题引擎、远程资产服务或后台配置页面。
- 不让图像模型生成“梦见樱町”中文文字，不把概念板裁切后直接当正式图标。
- 不把 Pixel2Motion 展示 HTML、回放控件、速度调节或 QA 面板嵌入正式站点，不把品牌动画扩展成占据头图背景主体的完整场景演出。
- 不修改文章正文作者内容来迁就主题，不增加正文装饰动画。

### 1.4 整体完成定义

只有同时满足以下条件才能声明完成：

1. 设计 §5.4-5.7、§6.2、§8.1、§9、§14 和 §15 的新增要求全部有实现、测试或真实浏览器证据。
2. 所有运行时资产均来自版本控制，生产页面不调用图像生成、抠图、字体 CDN 或第三方资产 API。
3. 字体按实际字符分片加载，浏览器不会为首屏一次下载四个完整 CJK 字体文件。
4. Logo、主题图标和状态图在浅色、深色、正式头图与棋盘格背景均无明显脏边。
5. 页面过渡不会因缓存命中被强制延迟，不会因错误、超时、快速连续导航或组件卸载留下遮罩。
6. 鼠标效果不替换系统光标、不拦截交互，并在触屏、输入、文本选择和减少动态效果环境关闭。
7. 作者允许格式可见，危险 HTML、事件属性、未知样式和危险 URL 不可执行。
8. `npm run typecheck`、`npm run lint`、`npm run test:run`、`npm run build` 和 `npm run format:check` 全部通过。
9. 真实浏览器完成 `1440×900`、`390×844`、减少动态效果和字体失败回退验收，无横向溢出、遮挡、文字裁切和明显布局跳动。

- 运行时为 React `19.2.7`、React Router DOM `7.18.1`、TanStack Query `5.101.2`、GSAP `3.15.0`、`@gsap/react 2.1.2`、DOMPurify `3.4.11`、Vite `8.1.3` 和 Vitest `4.1.10`。

- `src/styles/front.css` 集中承载前台视觉，但仍包含多处 Georgia、`Songti SC`、非零字距和 `clamp()` 字号。
- `src/main.tsx` 只加载 `@fontsource-variable/noto-sans-sc`；`index.html` favicon 为空，标题仍是“个人博客”。
- `FrontHeader.tsx`、`HomeView.tsx`、`ArticleListView.tsx`、`ArticleDetailView.tsx`、`FrontArticleMeta.tsx`、`CodeBlock.tsx` 和 `FrontAtmosphere.tsx` 直接导入 Lucide，尚无前台图标边界。
- `PageMotion.tsx` 只有新页面进入、文章行和单会话列车动画，没有离开/等待/收尾状态。

### 2.4 架构决策与假设

- 字体上游固定为 v3.750 的 Regular、Medium、Bold、Heavy，对应 `400/500/700/900`；不加载 ExtraLight、Light 和 Normal。
- 使用 `cn-font-split` 生成完整字集的 WOFF2 `unicode-range` 分片，不使用只扫描当前 JSX 文案的静态子集，因为文章正文和分类标签是动态内容。
- 生成式资产属于离线 authoring；Vite 构建只消费最终资产。上游原始响应、prompt、模型、时间和处理记录写入资产清单，但任何 API Key、授权头和本机绝对路径不得入库。
- 品牌、氛围和状态图使用生成式透明资产；菜单、关闭、搜索、方向箭头等熟悉功能继续使用 Lucide 几何，通过统一 `FrontIcon` 边界、颜色、线宽、状态装饰和动效消除“默认库样式”。存在 Lucide 图标时不手绘重复 SVG。
- 正式站名固定为“梦见樱町”，原“春日轨迹”视为早期临时名并从正式界面移除。站名由 `frontSite` 单一配置提供给导航、首页、浏览器标题和无障碍名称，未来站点配置 API 只替换数据来源。
- 首页品牌揭示只使用标题附近宽约标题 `18%-24%` 的图形标，不占头图背景主体。完整揭示按带版本号的 `sessionStorage` 状态每个浏览会话自动播放一次；存储失败直接显示完成态，桌面悬停只播放局部绽放。
- Pixel2Motion 固定为提交 `3824573d4e522c244d82b810fc5b85c7858f6b66` 并记录于资产 manifest。透明位图通过 BiRefNet 审计后再进行低复杂度 SVG 重建；花瓣、轨道和站点使用稳定语义 `id`。展示外壳留在 QA 产物，正式运行时只引入 `logo.svg` 和必要动画样式。
- 首页文章数据仍按现有 TanStack Query 流程加载。精选与最近文章在内容已存在于 DOM 后由 ScrollTrigger `once` 揭示；不使用 pin、scrub 或滚动触发请求，React 使用有作用域的 `useGSAP` 清理，字体/图片改变布局后刷新位置。
- 环形加载器的圆环、轨道、站点和进度弧使用 CSS/SVG 代码绘制，中心只引用简化 Logo；不会把动画烘焙成 GIF 或视频。
- 路由过渡只在“位置已变化且目标主查询没有可用数据、等待超过短暂防闪阈值”时显示。缓存命中、只发生背景刷新或查询在阈值内完成时只执行页面进入动画。
- 目标查询成功、空数据、业务错误或 `10s` 网络超时都视为页面状态已经可呈现，过渡必须收尾；遮罩始终 `pointer-events: none`。
- 现有动画偏好升级为前台共享运动偏好：关闭后停用环境花瓣和鼠标效果，页面等待仍保留静态可感知状态。兼容读取旧 `front-petals-enabled`，写入新键后删除旧键。
- 作者格式不通过宽泛的 `dangerouslySetInnerHTML` 直接信任。编辑器与公开阅读共享允许标签、颜色、背景色、对齐、宽度和 URL 规则；渲染前再次清理，未知 HTML 退化为安全文本或被移除。
- 仓库当前不存在 AGENTS 指定的 `test/unit/skills` 示例路径。正式功能实现开始时先按现有 `articleContent.test.ts` 的中文 Given-When-Then 形式建立行为骨架，并把缺失参考路径列为门禁说明；若执行环境提供了该路径，必须优先读取其中任一测试文件。

## 3. 生产架构

### 3.1 交付链路

```text
gpt-image-2 Mode A + 固定提示词
        -> 原始 PNG / 原始响应
        -> rembg + birefnet-general + Alpha Matting
        -> 透明边缘审计 / 尺寸规范化 / WebP-PNG 导出
        -> Pixel2Motion 固定提交
        -> 低复杂度语义 SVG / 路径审计 / Logo motion
        -> 静态拟合与确定性关键帧 / 最终帧一致性 QA
        -> src/assets/front + 资产清单
        -> Vite 哈希构建

寒蝉圆黑 v3.750 OTF
        -> 校验上游版本与摘要
        -> cn-font-split WOFF2 unicode-range 分片
        -> 字体 CSS / 许可 / 构建报告
        -> Vite 哈希构建
```

离线工具失败不得影响日常 `npm run dev` 和生产构建；已验收资产一旦提交，普通开发者无需安装 Python、ONNX 或字体拆分工具即可运行项目。只有重新生成资产时才执行工具链。

目录可以按实现时的最小职责微调，但必须保持“原始生成物、离线处理工具、运行时精选资产”三者分离，不把原始大图全部打进生产包。

### 3.3 字体加载契约

- `font.css` 只声明 `Chill Round Gothic` 的 `400/500/700/900`，每个分片带准确 `unicode-range` 和 `font-display: swap`。
- 固定界面、英文、数字和首页品牌字所在的关键分片允许预加载；动态 CJK 分片由浏览器根据页面字符请求。
- 前台根节点设置寒蝉圆黑，后台继续使用现有 Noto Sans SC，不让视觉升级扩散到管理端。
- 回退顺序以 `Noto Sans SC Variable` 为第一回退，确保度量接近；系统字体只处于最后安全位。
- 字号按桌面/移动明确断点切换，删除字体字号中的 `clamp()`；全部前台文本 `letter-spacing: 0`。
- 字体构建报告记录输入摘要、输出分片数量和大小。验收关注“按需请求且无整包下载”，不编造脱离真实产物的固定 KB 指标。

### 3.4 图标与品牌契约

- `FrontIcon` 提供受控名称、`16/24/32` 尺寸、状态和可访问属性；业务组件不再散落直接 Lucide 样式。
- 纯装饰图标统一 `aria-hidden`；独立图标按钮必须保留 `aria-label`/tooltip；可见文字已表达语义时不重复朗读。
- AI 资产必须具备透明 Alpha、固定安全留白和稳定视觉边界；状态图失败时仍显示文字与操作。
- Logo 文字永远由 HTML + 寒蝉圆黑渲染；favicon 和加载器只使用简化图形标。
- 功能图标在识别轮廓上复用 Lucide，在颜色、端点、状态底、花瓣/站点装饰和动效上统一；品牌资产不伪装成功能按钮。

### 3.5 首页品牌与滚动动效契约

- `frontSite.name` 的正式值为“梦见樱町”，导航、首页 H1、`document.title` 和无障碍名称从同一入口读取；旧临时名不得保留为运行时回退。
- 首页品牌开场由语义化 SVG 图形标和 HTML 文字共同组成；图形标宽约标题 `18%-24%`，只占标题安全区。完整时间线为轨道描线、站点点亮、樱花展开、标题淡入，最终帧必须等于静态 Logo。
- 带版本号的会话键只记录完整开场是否播放。首次进入播放一次，同会话返回直接显示完成态；会话存储异常同样显示完成态。桌面 hover 只调用独立局部绽放，触屏不注册 hover 动画。
- 精选文章以 section 为单一 ScrollTrigger：轨道线、站点章、主精选和次精选在同一 timeline 内按阅读顺序错峰，`once: true`，不设置 `pin` 或 `scrub`。
- 最近文章使用一次性批量触发，自下方约 `20px` 以 `70-100ms` stagger 淡入；快速滚动时批次立即完成到可读状态，不允许内容长期保持 `opacity: 0`。
- ScrollTrigger 只在文章 DOM 已存在后建立，不驱动 TanStack Query。使用 `useGSAP` scope 与组件 ref；卸载、依赖变化和路由切换自动 revert，字体与图片稳定后按需 `ScrollTrigger.refresh()`。
- 共享运动偏好关闭或 `prefers-reduced-motion: reduce` 时，Logo、精选与最近文章均直接显示最终态；静态轨道线、站点章和标题仍完整。

### 3.6 页面过渡状态契约

过渡由共享前台边界管理，状态至少包含：

```text
idle -> route-changed -> waiting-threshold -> visible-waiting
                                     |              |
                                     +-> ready -----+
                                                    v
                                                finishing -> idle
```

- `route-changed` 只由 `location.pathname + location.search` 变化触发。
- 主查询判定按当前路由映射 home/page/detail，只在 `fetchStatus=fetching` 且没有可展示数据时属于真实等待；分类、标签和后台刷新不触发整页过渡。
- 快速完成时取消阈值定时器，不挂载圆环；已显示时完成当前短收尾后卸载。
- 查询错误、超时、无效详情 ID 和空数据都进入 `ready`，由现有错误/空状态承担页面反馈。
- 连续导航以最新 location 为准，旧 Tween、定时器和订阅必须取消；过渡层不捕获指针和焦点。
- 减少动态效果或用户关闭运动偏好时，取消旋转、花瓣绕行和站点序列，只保留静态 Logo/圆环状态与即时页面切换。

### 3.7 鼠标与环境动画契约

- 只在 `(hover: hover) and (pointer: fine)`、页面可见、共享运动偏好开启且未命中 `prefers-reduced-motion` 时启用。
- 系统指针保持可见；花瓣跟随元素和粒子层 `pointer-events: none`。
- 输入、textarea、contenteditable、文本选择、拖动和窗口失焦期间暂停跟随与点击爆散。
- 使用固定数量对象池和 transform/opacity，禁止每次 pointermove 创建 DOM；高频位置更新使用现有 GSAP 能力或单个 requestAnimationFrame 合帧。
- 组件卸载、媒体条件变化、页面隐藏和运动偏好变化时清理事件、Tween、RAF、定时器和粒子。
- 现有环境花瓣数量调整为设计规定的桌面 `12-18`、移动 `5-8`，继续在页面隐藏时暂停。

### 3.8 作者格式与安全契约

- 编辑器与阅读器共享唯一的允许标签、文字色、背景色、对齐、宽度、表格跨行/跨列和 URL 协议规则。
- 前台支持编辑器真实产出的 `<span style>`、`<u>` 和复杂表格，不扩展 iframe、script、style、媒体、事件属性或任意 CSS。
- DOMPurify 清理发生在公开内容进入危险 HTML 边界之前；未知属性、未知颜色、背景图片、定位、脚本 URL 和事件处理器必须移除。
- 作者允许的 inline style 优先于 `.reading-content` 主题色；主题只提供默认值、字号、行高和间距。
- 代码块继续由 `CodeBlock` React 组件渲染，不因富文本链路失去复制按钮；H2/H3 ID 和目录行为保持稳定。
- 不信任“内容来自后台”作为绕过前端清理的理由，公开文章仍是跨信任边界数据。

### 3.9 数据库、后端、API、部署与兼容

- 数据库：不适用，本次不新增或修改数据字段、索引和迁移。
- 后端业务：不适用，文章公开规则、点赞、分类、标签和统计保持不变。
- HTTP API：不变，不新增 Swagger/OpenAPI 工作；仍使用现有 `/api/home`、`/api/articles` 和 `/api/articles/{id}`。
- 安全：唯一新增信任边界是前台展示作者允许 HTML；由共享白名单和 DOMPurify 防御，不改变后端存储契约。
- 部署：静态包会增加字体分片和品牌资产。回滚只需回退前端提交/静态构建，不涉及数据恢复。
- 兼容：不支持脚本、WebGL 或复杂滤镜的环境仍获得静态 Logo、文字和现有页面功能；动画与装饰失败不得阻断导航和阅读。
